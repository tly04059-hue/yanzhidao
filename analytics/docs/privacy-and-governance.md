# 隐私、权限与治理

## 1. 隐私原则

1. 行为埋点不强制微信登录。
2. 行为事件默认只保存匿名用户 ID。
3. 手机号、微信号只保存在留资数据中，不写入普通行为事件。
4. 不采集身份证号、通讯录、精确定位、设备敏感信息。
5. 隐私政策中应说明会收集浏览、点击、停留、分享、留资等数据用于产品优化。

## 2. 数据关联

用户提交留资后，可以用以下字段把行为链路和留资关联：

- `anonymous_user_id`
- `session_id`
- `source_channel`
- `recommended_path`
- `recommended_schools`
- `last_page`

普通行为事件不直接保存手机号或微信号。

## 3. 权限建议

| 角色 | 可看数据 |
| --- | --- |
| 管理员 | 全量看板、事件明细、留资数据。 |
| 运营 | 趋势、漏斗、内容热度、渠道效果。 |
| 顾问 | 留资列表和用户推荐结果，不看完整事件流。 |
| 开发 | 测试环境事件、接口错误、埋点失败率。 |

## 4. 数据保留建议

| 数据 | 建议 |
| --- | --- |
| 原始行为事件 | 保留 180 天到 1 年。 |
| 每日汇总指标 | 长期保留。 |
| 留资数据 | 按业务和隐私政策要求保留。 |
| 测试数据 | 可定期清理，不进入经营指标。 |

## 5. 异常监控

后续看板应提示：

- 今日事件量为 0。
- PV 正常但留资为 0。
- 某关键页面退出率突然升高。
- 埋点接口错误率升高。
- 某版本事件字段缺失率升高。
