# schoolData 院校库目录说明

`schoolData` 是研知道项目的院校库主目录，所有院校相关数据都应在这里沉淀。

当前阶段先不移动已有运行文件，避免影响小程序和现有 Web 页面。后续新增数据按下面分层放置。

## 目录结构

| 路径 | 层级 | 用途 |
| --- | --- | --- |
| `sourceData/` | L0 源数据层 | 原始 Excel、CSV、网页抓取结果、人工整理表。只追加，不覆盖。 |
| `standardized/` | L1 标准化层 | 清洗后的标准数据，字段稳定，供脚本、推荐策略、AI 检索复用。 |
| `publish/` | L2 发布层 | 小程序可直接读取的数据，只放审核通过且完整度达标的数据。 |
| `schemas/` | 字段标准 | 标准化层和发布层的 JSON Schema。 |
| `templates/` | 示例模板 | 给同事和脚本参考的单条数据样例。 |
| `mappings/` | 清洗映射 | 源数据字段到标准化 JSON 字段的映射规则。 |
| `data.json` | 当前运行数据 | 现有 MPA/MBA/MEM 院校展示数据，暂时保留。 |
| `dx_data.json` | 当前运行数据 | 现有党校数据，暂时保留。 |
| `yanzhidao_schooldata.*` | 工具页面 | 院校库本地 Web 展示页。 |

## 数据原则

1. 源数据不直接给小程序读。
2. 标准化层保留尽可能完整的数据。
3. 发布层只保留前台需要、且已审核的数据。
4. 每次更新必须能追溯到来源文件、来源行、更新批次。
5. 不为了适配前台展示而破坏原始数据颗粒度。

## 一期目标

一期先确保小程序可展示的基础字段完整度超过 80%：

- 学校名称
- 项目类型
- 省份/城市
- 学费
- 学制
- 学习方式
- 招生人数
- 是否接受大专报考
- 上课方式
- 上课地点
- 最新分数线
- 是否调剂
- 退役计划/少骨计划
- 学校/项目简介
- 录取分析或复试信息

二期和三期重点补年度复试考情、一志愿录取分析、调剂录取分析、分数段分析等深度字段。
