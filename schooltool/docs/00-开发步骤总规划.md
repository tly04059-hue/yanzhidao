## 总览

```
Step 1  产品定义        → 用户是谁、解决什么问题、核心价值
Step 2  页面与流程设计   → 有哪些页面、页面关系、每页干什么
```

## Step 8：后端开发

**目标：** 实现推荐引擎 API、数据存储、管理后台

### 8.1 开发顺序

```
Phase 8a：基础设施
  - 项目初始化（Express + SQLite/MySQL）
  - 数据库建表、数据导入
  - API 框架搭建

Phase 8b：核心 API
  - POST /api/start — 创建访客
  - POST /api/answer — 记录答题
  - POST /api/recommend — 推荐引擎（决策树 + 匹配 + 排除规则）
  - POST /api/lead — 留资
  - GET  /api/schools — 院校列表
  - GET  /api/schools/:id — 院校详情

Phase 8c：安全与防护
  - API 签名 + 时间戳校验
  - 频率限制
  - 数据埋点

Phase 8d：管理后台（Phase 2）
  - 线索列表 + 画像查看
  - 案例库管理
  - 数据看板
```

### 8.2 完成标准

- [ ] 推荐 API 全组合测试通过
