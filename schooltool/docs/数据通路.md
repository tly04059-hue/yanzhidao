## 4. apply 前检查 · 脱敏出口检查（强制门槛）

**入口位置（2026-05-27 ww 钦定调整）**：

| 旧规则 | 新规则 |
|---|---|
| "派生 JSON 生成时跑 check_privacy" | "派生到选校工具消费层（`data/derived-from-context/*.json`）前一步跑 check_privacy" |
| M8 csv 必须是脱敏对照库 | M8 csv 是全量内部 SSOT，不脱敏 |

**含义**：
- M8 SSOT 内部不脱敏（含真名、详细地址、单位全称）
- 派生脚本（如 derive_cases_json.py）输出到选校工具消费层前**必须**完成 4 层脱敏
- 5 个敏感 JSON 仍受 `数据脱敏规则.md` 4 层框架约束：
  - cases.json · 🔴 最高敏感
  - students_map.json · 🟡 中
  - peer_profile_cache.json · 🔴 最高
  - path_options.json · 🟢 低
  - rules.json · 🟢 低
