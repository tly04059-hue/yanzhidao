# 选校诊断工具 · 院校库与路径库设计

> 版本：v0.1 | 日期：2026-05-05
> 用途：统一内容录入范围、院校核验顺序、路径库结构
> 前置：`02-产品需求文档-V1.md`、`04-数据字典.md`

---

## 一、文档目的

本文件解决两个问题：

1. **V1 到底先录哪些学校和路径**
2. **每个页面涉及的院校信息，按什么顺序核验和补齐**

它不是招生简章汇总，也不是学校百科，而是 V1 内容建设的执行主表设计文档。

---

## 二、V1 内容库拆分

V1 先拆成两层：

### 2.1 路径库

先回答“有什么路可走”。

建议首批路径：

1. 四川党校
2. 重庆党校
3. MPA
4. MBA
5. 教育硕士
6. 法学相关非全路径
7. 先升本再读研
8. 暂不建议立即报考

### 2.2 院校项目库

再回答“具体可报哪些项目”。

院校项目库不追求一开始全量，而按咨询频次和结果页优先级覆盖。

---

## 三、V1 首批覆盖原则

### 3.1 按咨询频次优先

先覆盖顾问最常被问到的学校和路径，而不是先追求“大而全”。

### 3.2 按结果页优先

结果页会被推荐到的项目，必须优先入库并核验。

### 3.3 按页面曝光优先

首页、了解页、结果页、院校详情页会直接展示的内容，优先核验。

### 3.4 按风险高低优先

以下信息优先核：

- 报考条件
- 是否需要党员
- 是否认可用于特定目标
- 学费与学制
- 特殊限制条件

---

## 四、V1 首批路径库建议

| 路径 | 当前定位 | 是否 V1 必做 | 备注 |
|------|----------|---------------|------|
| 四川党校 | 主力路径 | 是 | 重点建设 |
| 重庆党校 | 主力路径 | 是 | 与四川党校构成对比 |
| MPA | 主力替代路径 | 是 | 高频比较对象 |
| MBA | 补充路径 | 否，建议 V1.1 | 先保留结构 |
| 教育硕士 | 垂直路径 | 否，建议 V1.1 | 教育系统适配 |
| 法学相关非全 | 垂直路径 | 否，建议 V1.1 | 法检公安延展 |
| 先升本再读研 | 阻断/过渡路径 | 是 | 大专用户必须支持 |
| 暂不建议立即报考 | 风险保护路径 | 是 | 用于结果阻断与风险提示 |

---

## 五、V1 首批院校项目范围建议

### 5.1 必须先完整核验的项目

- 四川党校全部 V1 展示项目
- 重庆党校全部 V1 展示项目
- 结果页明确会推荐的 MPA 项目
- “先升本再读研”承接页涉及的升本路径说明

### 5.2 可先占位的项目

- MBA
- 教育硕士
- 法学类非全
- 其他国民教育序列延展项目

这些可以先保留页面入口，但不急着做大规模完整库。

## 六、页面与内容库映射

| 页面 | 需要调用的内容 | 优先级 |
|------|----------------|--------|
| 首页 | 核心路径标签、信任数据、产品说明 | 高 |
| 了解页 | 路径说明、学历价值内容、案例摘要 | 高 |
| 路径对比页 | 路径库对比字段 | 高 |
| 院校列表页 | 院校项目基础字段 | 高 |
| 院校详情页 | 单个项目完整字段 | 高 |
| 结果页 | 路径库 + 院校项目 + 风险提示 + CTA | 最高 |
| 备考页 | 备考方案和承接内容 | 中 |

---

## 七、院校核验流程

每个项目按以下顺序核验：

1. 是否仍在 V1 覆盖范围
2. 基础字段是否完整
3. 信息来源是否为官方或可追溯来源
4. 是否影响推荐逻辑
5. 是否已写入结果页或院校详情页
6. 是否标注最近核验时间

---
