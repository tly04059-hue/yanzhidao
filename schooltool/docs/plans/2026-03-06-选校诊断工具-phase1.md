# 选校诊断工具 Phase 1 实施计划

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** 构建一个可用的H5选校诊断工具，用户答8道题后得到个性化推荐+案例+引导加微信。

**Architecture:** 前端Vue3 CDN + Vant移动端UI（无构建步骤），后端Express API + SQLite。前后端分离，Nginx反代。本地开发后部署到阿里云。

**Tech Stack:** Node.js 22 / Express / better-sqlite3 / Vue 3 CDN / Vant 4 CDN

### Task 2: 数据库 + 数据初始化

**Files:**

---

### Task 3: 推荐引擎

**Files:**
- Create: `server/engine.js` — 核心推荐逻辑
- Create: `server/rules.js` — 排除规则+提示规则

**输入：** 用户8道题的答案对象
**输出：** { primary, alternative, case, warnings, key_info }

逻辑：排除规则 → 专业匹配 → 院校匹配 → 案例匹配（5维度加权）

---

### Task 4: API路由

**Files:**
- Create: `server/routes.js`

**接口：**
- `POST /api/start` — 创建访客，返回visitor_id
- `POST /api/answer` — 记录每道题答案
- `POST /api/recommend` — 提交全部答案，返回推荐结果
- `POST /api/lead` — 提交微信号，关联画像
- `GET /api/stats` — 后台数据统计（需简单鉴权）
