# PRD 文案统一策划原则

> 版本：v0.1 | 日期：2026-05-19
> 定位：选校诊断小程序 PRD 文案的**正向策划纪律**。所有文案修改前必读、修改时遵循、修改后过校对清单。
> 关联：02-PRD-V1.md（文案落地处）· 专项/03-内容审核清单.md（执行层权威）· 09-数据来源与上下文库映射规则.md（派生层架构）

---

## 一、为什么需要这份原则

PRD v0.1 → v3.x 一路是"原型先行 → PRD 反哺"——文案多数从 HTML 原型里抠出来，再补背景。这种链路有 3 个问题：

1. **来源不可追溯**：句子怎么来的、为什么这样写、改了会不会破坏立场，没人能回答
2. **声音不统一**：同一个 PRD 里能同时出现 ww 第一人称 / 机构官方口吻 / 焦虑营销腔调
3. **改一处崩一片**：上下文价值观层一更新，PRD 不知道哪些句子要联动改

本文档要建立的是反向链路：

```
上下文库（事实层 + 价值观层）
        ↓ 正向策划
PRD 文案（带来源指针、过 3 元规则、合信任公式）
        ↓ 原型实现
HTML / 小程序 UI
```

PRD 是文案的权威，原型只是 PRD 的实现样张。

## 二、文案的 3 层来源

PRD 里**每一段对外文案**都必须能定位到下面三层来源中至少一层。无来源 = 不能上线。

### L1 事实层（数据与字段）

- **做什么**：所有数字、政策、院校信息、过考率、字段定义，必须来自可追溯源
- **不做什么**：不凭印象写、不用未经校对的旧版数据、不让 AI"合理推导"出数字
- **来源池**：政策文件原文 / M9 院校库（`02-市场与用户/院校库/`）/ M8 学员案例库 / 04 数据字典 / 专项/03 §六过考率权威口径
- **正例**："四川党校 25 年招生 600 人，整体录取率约 9.6%（含弃考）" — 引自 03 §一.1
- **反例**："党校录取率不到 10%" — 没指明四川/重庆、没指明含不含弃考、没注明年份

### L2 立场层（语调与措辞）

- **做什么**：决定句子的姿态——是 ww 第一人称、是判断书式的专业语气、是不催不诱的服务节奏
- **不做什么**：不写空洞品牌承诺、不用机构官方口吻、不做焦虑营销
- **来源池**：上下文价值观层 §3 信任方程式 / §3.7 AI 时代立场 / §五 团队判断语言库；业务必读 §5 业务方向核心口径（§5.1 防御性产品 / §5.5 六个不做）；经营战略总纲 §3.1.2 起点承诺三层结构 / §5.2 专业立场公理（含第 ⑥ 类抹除客群场景）
- **正例**："如果你现在动机不清晰，我建议先不要报名" — 体现"敢说不"+ ww 第一人称
- **反例**："研知道致力于为您提供专业的考研辅导服务" — 机构官方口吻，违反价值观层 §5.3

### L3 执行层（字句与禁用词）

- **做什么**：句子级的合规——禁用词扫描、ER1/ER2/ER3 过滤、防御性表述检查
- **不做什么**：不出现 §5.4 禁用词、不打主力客群标签、不催单不强分享
- **来源池**：专项/03 §零 6 组硬条款（ER1/ER2/ER3 + §0.4 禁用词清单 + §0.5 反面禁止 + §0.6 防御性表述纪律）
- **正例**："保留可能性 / 不掉队 / 占位" — §0.6 防御性表述
- **反例**："助力你考研路上一往无前" — 命中 §0.4 空洞承诺（助力）+ 命中 §0.6 进攻性话术

**三层关系**：L1 给砖头、L2 给图纸、L3 给质检尺。三层都过了才能砌墙。

## 三、3 条元规则与具体执行

3 条元规则来自 03 §零，是任何 PRD 文案上线前必过的硬门槛。

### ER1 · 不打"主力客群"标签

**定义**：文案不得含"主力 / 最强 / 最集中 / 主要面向 / 谁占多数"等暗示概率分布的措辞——主力客群只能由客户管理客户资料的清单分布给定，不能由抽检 / AI 推导 / 惯性判断给定。

**5 个正向措辞**：
1. "遇到该场景的客户全部适用"
2. "对应场景"
3. "符合 X 条件的人"
4. "X 情况下的常见路径"
5. "本工具覆盖以下场景"

**5 个反例措辞**：
1. "主力客群是 30-35 岁公务员"
2. "我们最强的服务方向是 MPA"
3. "大部分学员都选择党校"
4. "主要面向体制内人群"
5. "本工具最适合 X 类用户"

**应用场景**：首页副标题 / 测一测引言 / 结果页推荐理由 / 案例匹配说明 / 关于研知道页

### ER2 · 政策口径用"可能性和判断方向"

**定义**：所有政策结论用"可能性 + 判断方向"措辞，不做确定的因果论断。政策事实可以确定，政策对个人的作用不能确定。

**5 个正向措辞**：
1. "政策依据 / 政策方向 / 政策窗口"
2. "可能的应用场景"
3. "符合该方向的可能性"
4. "⚠️ 具体执行视个人情况"
5. "判断空间 / 决策空间"

**5 个反例措辞**：
1. "一定能落编"
2. "肯定可以遴选"
3. "必然升职加薪"
4. "保证调任成功"
5. "100% 符合新政策"

**应用场景**：结果页政策提示 / 加载页政策卡 / 了解 Tab 政策大背景段 / 判断书风险与边界段

### ER3 · 不催单 + 不强分享 + 圈子重合保护

**定义**：三个硬约束合一——不催单（时间紧迫感不来自机构）、不强制公开传播（用户自决是否发朋友圈）、不让朋友圈尴尬（默认私域承接，公域裂变慎用）。

**5 个正向措辞**：
1. "保存到手机 / 私聊朋友 / 分享给信任的人"
2. "你来定节奏，需要时再问"
3. "想看完整版可以加微信"
4. "分享海报"（按钮明示三选一：给自己 / 给朋友 / 给信任的人）
5. "随时可以来咨询，没有限期"

**5 个反例措辞**：
1. "今天最后一天 / 立即报名 / 倒计时"
2. "分享朋友圈解锁完整报告"
3. "转发到群解锁权益"
4. "名额仅剩 X 个"
5. "马上就要截止了"

**应用场景**：CTA 按钮文案 / 留资弹窗 / 分享好物中心 / 结果页底部承接 / 加企微引导

**圈子重合保护的额外注意**：研知道服务的体制相关人群，单位 / 部门 / 系统内人际关系敏感。任何让"考研这件事被同事看见会尴尬"的设计 = 违反 ER3。判断方法：把这个文案换成"如果是用户的同事看到，他会不会觉得 awkward"——会，就改。

## 四、信任公式在 PRD 文案中的落地

**信任 = (可信度 × 可靠度 × 亲密感) ÷ 自我导向感**（价值观层 §3.3）

PRD 文案要逐句问自己：这句话是在抬哪个变量？

### 可信度↑（让用户相信）

- **数据可追溯**：每个数字标年份、口径、来源（如"四川党校 40.4%（25 年辅导口径）"）
- **案例真实**：用真实姓 + 名字脱敏（X 同学），单位到子类、地域到市级（03 脱敏规则）
- **政策来源清晰**：政策段附文号 / 部门 / 发布日期
- **正例**："2026 国考首次放宽至 38 岁（M2 §E.0），川渝省考已跟进"
- **反例**："最新政策对在职人群非常友好"

### 可靠度↑（让用户安心）

- **不催不诱**：CTA 不带时间压迫感
- **留退路**：每个结果页给"如果不符合可以怎么办"
- **节奏由用户定**："你来定节奏 / 想了解再问 / 随时可以来"
- **正例**："想看更详细的 1 对 1 诊断，可以加我们顾问微信"
- **反例**："立即领取你的专属诊断报告，名额有限"

### 亲密感↑（让用户感觉"研知道懂我"）

- **地域化语言**：四川 / 重庆严格分开，不合并表述；用"川渝在职"不用"全国学员"
- **真实场景**：用 M5 触发场景的原话（"同事在读研" / "遴选公告要求硕士"）而不是宽泛描述
- **同行案例**：同体制、同岗位、同年龄段的具体案例（M8 案例库 K-NN ≥3 隐私门槛）
- **正例**："和你同样在事业编专技岗、想评副高的张同学，最终选了 MPA"
- **反例**："众多体制内同学的共同选择"

### 自我导向感↓（不让用户觉得机构有所图）

- **不强分享 / 不公域裂变 / 不利益换信息**（ER3 落地）
- **报价被动**：被问到才报，不主动推
- **不留"销售钩子"**：结果页不给"输入手机号查看完整方案"这种条件性内容
- **正例**："这是完整诊断，没有藏后面的部分"
- **反例**："留下微信号查看完整版" + "扫码解锁 5 大独家政策解读"

**判断手势**：每段文案问"这句话让用户觉得研知道在为他打算，还是在为自己打算？" — 前者放行，后者改。

## 五、禁用词清单 + 替代措辞表

完整禁用词来自 03 §0.4 + §0.5 + §0.6。任何 PRD 文案修改后必须过本表扫描。

| 类别 | 禁用词 | 合规替代 |
|---|---|---|
| 空洞承诺 | 助力 / 赋能 / 保驾护航 / 一站式 / 全方位 | 说具体做什么（如"给出 3 个路径选项"） |
| 绝对化 | 保过 / 必过 / 100% / 最好 / 唯一 | 用具体数据（如"25 年辅导过考率 40.4%"） |
| 机构口吻 | 我们致力于 / 我们的使命 / 我们的愿景 | ww 第一人称 / 第二人称"你" |
| 品牌自嗨 | 研知道学员 / 研知道家人 / 学姐学长 | 直呼"你" / "X 同学" |
| 焦虑推手 | 再不 X 就来不及了 / 别人都已经 / 不抓住就错过 | 客观描述窗口期（"38 岁是 2026 国考新放宽的上限"） |
| 推卸判断 | 看个人情况 / 仁者见仁 / 各有利弊 | 给明确判断 + 理由 |
| 进攻性话术 | 升职加薪 / 影响力变大 / 改变命运 / 走上人生巅峰 / 实现梦想 | 保留可能性 / 不掉队 / 占位 / 防御性资产 / 确定性投资 |
| 主力客群暗示 | 主力 / 最强 / 最集中 / 谁占多数 | 遇到该场景的客户全部适用 / 对应场景 |
| 必然预测 | 一定 / 肯定 / 必然 / 保证 | 政策依据 / 判断方向 / 可能性 |
| 催单 | 今天最后 / 名额仅剩 / 立即 / 倒计时 / 马上截止 | 随时可以咨询 / 想了解再问 |
| 强分享 | 分享朋友圈解锁 / 转发到群解锁 / 分享得礼包 | 保存到手机 / 私聊朋友 / 分享海报（三选一明示） |

## 六、文案场景库（PRD 每一类场景的统一调性）

按 PRD 页面类型分类，给每类的文案调性指引。所有新增/修改文案先定位到对应场景，再写。

### 6.1 入口页文案（首页 / 测一测入口 / 关于研知道）

- **调性**：扎实 + 克制，3 秒说清"是什么 / 给谁 / 怎么用"
- **必含**：业务必读 §一对外品牌承诺（"不骗人真服务，稳定提高过考率"）+ 聚焦川渝
- **禁止**：空洞口号 / 流量话术 / 焦虑钩子
- **正例**："川渝体制内在职读研选校诊断 · 回答几个问题，3 分钟给出适合你的方向"
- **反例**："开启你的考研逆袭之旅！"

### 6.2 答题页文案（题目 / 选项 / 即时提示）

- **调性**：中性 + 不引导，让用户回答"我是谁"，而不是回答"你期待我是谁"
- **必含**：每题用 M5 触发场景的真实措辞 / 选项穷举不暗示偏好
- **禁止**：选项里暗示"标准答案" / 用判断性形容词（"积极的 / 进取的"）
- **正例**：Q6 核心目标 · "本单位晋升 / 遴选到上级机关 / 职称评审 / 想转公务员 / 防御性学历"
- **反例**：Q6 改成"你最想通过考研获得的成就是？" — 把工具问题变成鸡汤问题

### 6.3 结果页文案（推荐理由 / 路径含义 / CTA）

- **调性**：判断书式 + 给推导链，不给结论 + 不给口号
- **必含**：基于什么条件 → 做了什么分析 → 给什么建议 → 为什么（经营战略总纲 §3.1.2 判断书定义）
- **禁止**：单句结论 / 营销修辞 / 强制下一步
- **正例**："你的情况：四川 · 公务员 · 31-35 岁 · 党员 · 目标本单位晋升 → 首选四川党校政治学，备选 MPA。原因：①党员 + 公务员 = 党校系统对口；②本单位晋升不强制要求双证"
- **反例**："恭喜你！为你匹配到了最佳选择 — 四川党校"

### 6.4 承接页文案（加企微 / 备考方案）

- **调性**：被动 + 留退路，让用户主动发起下一步
- **必含**：明示"可加 / 可不加" + 加了之后会发生什么 + 不催不锁
- **禁止**：分享换权益 / 限时优惠 / "立即"句式
- **正例**："想要更详细的 1 对 1 诊断 → 加顾问微信。顾问会在 24 小时内联系你，不发广告"
- **反例**："立即留资解锁完整报告 + 8 大独家政策" + "今天加微信送资料包"

### 6.5 信息页文案（过考率说明 / 数据来源 / 使用说明）

- **调性**：透明 + 自查，给用户"看穿事情"的能力
- **必含**：所有数据明示口径（辅导 vs 自然 / 含弃考 vs 实考 / 四川 vs 重庆 / 初试 vs 复试）
- **禁止**：合并表述 / 模糊口径 / 省略来源
- **正例**："四川党校 25 年辅导过考率 40.4%（研知道辅导班 1,123+ 同学样本）· 市场自然过考率约 9.8%（综合评估，非具体院校准确值）"
- **反例**："党校过考率 40% +"

## 七、文案校对 checklist

PRD 任何新增 / 修改文案，按本清单逐项过。命中任何一项 NO → 不能上线。

| # | 检查项 | 标准 |
|---|---|---|
| 1 | **来源追溯** | 这句话出自上下文库哪个文档哪段？无 → NO |
| 2 | **L1 事实合规** | 数字 / 政策 / 院校 / 字段是否引自 03 §六、M9、04 数据字典等权威源？ |
| 3 | **L2 立场合规** | 语调是否符合价值观层 §3 / §5 + 业务必读 §5？ |
| 4 | **L3 执行合规** | 是否过了 ER1/ER2/ER3 + §0.4 禁用词 + §0.5 反面禁止 + §0.6 防御性表述？ |
| 5 | **信任公式定位** | 这句话抬哪个变量？（可信度+ / 可靠度+ / 亲密感+ / 自我导向感-）说不清 → NO |
| 6 | **禁用词扫描** | 第五节 11 类禁用词全过一遍 |
| 7 | **川渝口径区分** | 涉及党校 / 院校 / 数据时，四川 / 重庆是否严格分开不合并？ |
| 8 | **辅导 vs 自然口径** | 涉及过考率时，是否明示是辅导口径还是自然口径？ |
| 9 | **管综初试 → 复试递进** | 涉及管综数据时，是否明示初试和复试是递进关系？ |
| 10 | **防御性表述纪律** | 是否避免"升职加薪 / 改变命运"等进攻性话术？ |
| 11 | **圈子重合自查** | "如果用户的同事看到这句话，会不会觉得 awkward"？ |
| 12 | **不催不强** | CTA 是否避免时间压迫 / 强制分享 / 锁内容？ |
| 13 | **ww 第一人称一致性** | 立场判断类句子是不是 ww 在说话？机构口吻 → NO |
| 14 | **价值观层 4 项底色滤镜** | 扎实 / 不骗人真服务 / 敢说积极发心 / 不留尾巴 — 全过？（价值观层 §5.7） |
| 15 | **PRD 上下游一致** | 改了这句话，结果页 / 政策段 / 案例段 / 数据字典是不是要联动改？ |

## 八、维护流程

### 8.1 PRD 改文案时

1. 找到要改的段落，定位到第六节场景库哪一类
2. 按第二节 3 层来源（L1/L2/L3）找源头
3. 写草稿
4. 过第七节 15 项 checklist
5. 改 PRD 时同时在改动记录里写明"来源 + 信任公式定位 + 哪个 ER 通过"
6. 联动检查 09 派生层映射清单是否要同步刷新

### 8.2 上下文库更新时（反向触发）

上下文价值观层 / 业务必读 / 经营战略总纲 / 03 内容审核清单更新后：

1. 查 09 §五 sync_status 检测器，看哪些派生文件被标 stale
2. 同时手动扫 PRD 里**直接引用**这些文档的段落（如品牌承诺 / 过考率表 / 政策段 / 防御性表述例句）
3. 评估 PRD 文案是否要联动改
4. 改完后过第七节 checklist
5. 改动记录里注明"由 XX 文档 YY 段更新触发"

### 8.3 与 09 派生层的关系

PRD 文案分两类来源：

| 类型 | 特征 | 来源 |
|---|---|---|
| **hardcode 文案** | 框架性、低频改、与产品形态绑定 | PRD 直接写（首页标题 / 答题流程说明 / 信息架构标签 / 隐私协议） |
| **派生文案** | 数据驱动、规则驱动、按上下文库版本走 | 09 派生层（题目内容 / 选项 / 推荐理由 / 案例 / 政策提示 / 判断书 / 分享好物） |

**判断方法**：这句话改的频率取决于什么？

- 取决于产品决策 → hardcode（写进 PRD）
- 取决于上下文库 → 派生（写进 `data/derived-from-context/*.json`，PRD 只规定字段和契约）

PRD 不重复派生文案的内容，只规定它们的**字段、契约、调性**。

### 8.4 版本管理

- **小改**（措辞调整 / 单段文案优化）：PRD 末尾改动记录加一行，注明触发源
- **中改**（一个场景的整段文案改写 / 新增一类场景）：PRD 升小版号（v3.x → v3.(x+1)），改动记录注明 ER 通过情况
- **大改**（涉及品牌承诺 / 主线 CTA / 信任公式落地策略调整）：PRD 升大版号（v3 → v4），必须 ww 校对入库；同步刷新本文档（PRD-文案统一策划原则）
