## 一、V1 目标

帮助川渝体制内用户完成学历路径决策的全链路接入 — 从"我要不要读研"到"我该走哪条路"到"下一步做什么"。

### V1 要接住的三类用户

| 用户心理             | 产品入口     | 核心体验           |
| ---------------- | -------- | -------------- |
| 犹豫期："我是不是也该读个研？" | "了解"Tab  | 看政策、看案例，理解学历作用 |
| 决策期："我该考什么？"     | "测一测"Tab | 答题 → 个性化方案     |
| 行动期："我决定了，怎么备考？" | 首页备考入口   | 备考方案、资料、联系顾问   |

### V1 业务目标

- 复合型低门槛引流产品：浏览式留存 + 答题式转化
- 降低重复咨询成本：顾问拿画像而不是空白微信号
- 建立品牌信任：政策透明 + 过考率公开 + 不卖课卖判断

## 三、设计原则

### 3 步原则

从打开到获取关键信息，控制在 3 步以内。

| 用户目标 | 步骤 | 步数 |
|---------|------|------|
| 做选校诊断 | 首页/测一测→答题→结果 | 3 |
| 了解本系统学历作用 | 首页→了解→选系统 | 3 |
| 路径对比（党校 vs MPA） | 了解→路径对比 | 2 |
| 查院校信息 | 首页→院校查询→详情 | 3 |
| 领取备考资料 | 首页→资料领取→填微信号 | 3 |
| 联系顾问 | 首页→客服 | 1 |

### 操作深度控制

| 步数 | 原则 | 示例 |
|------|------|------|
| 第 1 步 | 一眼看懂，一个动作 | 首页→点"我还在犹豫" |
| 第 2 步 | 选项极少，选了就走 | 了解 Tab 四宫格选一个维度 |
| 第 3 步 | 可以有信息量，辅助判断 | 浏览详情页（政策+案例+路径） |
| 第 4 步 | 行动收口 | 留资 / 分享 / 测一测 |

### 用户特征适配

- 主要用户 35 岁+：字要大（正文 16px+）、按钮要大（96rpx+）
- 体制内人群：稳重、可信、不花哨，政策和数据说话
- 注意力有限：能少就少，干扰最小化
- 95% 从微信打开：移动端优先，PC 不适配

### 内容原则

- **不做鸡汤，只做政策解释和案例匹配**
- 所有"为什么读研"的回答必须基于政策依据或学员数据
- 所有推荐必须给出依据（政策文件号 或 学员统计）

**每个浏览详情页的统一结构（5 段）：**

1. **政策依据** — 文件 + 条款 + 作用
2. **实际作用** — 晋升 / 涨工资 / 遴选三类硬作用
3. **同行数据** — 基于同系统学员的统计分布
4. **典型学员** — 1-2 个代表性案例（可展开查看更多）
5. **推荐路径** — 首选 + 备选
6. **底部 CTA** — "测一测我适合哪条"

**六种浏览维度的内容差异：**

| 维度 | 分类选项 | 内容侧重 |
|------|---------|---------|
| 按系统 | 党政/财税/公检法/教育医疗/乡镇/国企/民族地区 | 该系统的晋升机制和学历作用 |
| 按目标 | 本单位晋升/遴选/转公/职称/防御 | 该目标对应的硬政策要求 |
| 按学历 | 全日制本科/非全日制本科/大专 | 该起点下的可选路径和陷阱 |

## 八、技术要求概要

### 前端

- 框架：微信小程序原生 或 uni-app（H5+小程序双端）
- 推荐逻辑**不放前端**，所有计算走后端 API
- "了解"Tab 的内容页数据走 API 动态加载（防爬）
- 不做注册登录，用微信 openid 静默获取

### 后端

- Node.js (Express) + SQLite/MySQL
- 已有阿里云 ECS 服务器

### 核心 API

| 接口 | 方法 | 说明 |
|------|------|------|
| /api/start | POST | 创建访客，记录来源 |
| /api/answer | POST | 记录每题答案 |
| /api/recommend | POST | 提交全部答案，返回推荐（含 path_meaning） |
| /api/lead | POST | 提交微信号 |
| /api/schools | GET | 院校列表 |
| /api/schools/:id | GET | 院校详情 |
| /api/browse/system/:id | GET | 按系统浏览详情 |
| /api/browse/goal/:id | GET | 按目标浏览详情 |
| /api/browse/education/:id | GET | 按学历浏览详情 |
| /api/browse/age/:id | GET | 按年龄浏览详情 |
| /api/cases | GET | 案例库（支持筛选） |

### 防护

- API 签名 + 时间戳校验
- 频率限制：同一 IP 10 分钟内最多 3 次测试
- 案例文本通过 API 动态加载
- 推荐逻辑参数不暴露给前端
