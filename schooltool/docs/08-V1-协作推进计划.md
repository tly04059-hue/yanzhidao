# 选校诊断工具 · V1 协作推进计划

> 版本：v0.1 | 日期：2026-05-05
> 用途：用于产品、内容、前端、后端并行协作的轻量推进表

---

## V1 协作推进总表

| 阶段 | 时间建议 | 你 / 产品 | 内容 / 数据 | 前端 | 后端 | 产出 |
|------|----------|-----------|-------------|------|------|------|
| Phase 1 锁范围 | 第 1 周 | 收紧 V1 页面范围；确认必做 / 可延后 / 暂不做；定结果页必须输出项 | 确定 V1 首批路径和院校范围；标记待核内容 | 先看页面结构，准备页面壳 | 先看数据对象和接口范围 | 锁定 V1 范围 |
| Phase 2 定数据 | 第 1-2 周 | 确认每个页面需要哪些数据字段 | 补院校库、路径库、来源和更新时间；先核结果页相关内容 | 用假数据搭页面 | 建模型和基础接口骨架 | 字段和数据基线 |
| Phase 3 定逻辑 | 第 2 周 | 确认首推、备选、阻断、风险提示规则 | 补推荐理由和风险提示文案 | 接假数据完成题流、结果页交互 | 实现核心推荐逻辑 | 规则初版可跑 |
| Phase 4 前后端并行 | 第 2-3 周 | 按页面验收：首页、测一测、结果页、院校详情页 | 持续核页面所需数据 | 完成页面壳、状态、接口接入 | 完成接口、规则执行、结果返回 | 可联调版本 |
| Phase 5 联调修正 | 第 3-4 周 | 逐页验页面、数据、逻辑是否一致 | 修正错误数据和缺失内容 | 修交互和展示问题 | 修接口和规则问题 | 可体验版本 |
| Phase 6 上线前收口 | 第 4 周 | 确认 MVP 是否达到发布标准 | 完成第一轮重点内容复核 | 修移动端细节和异常态 | 补监控、容错、基础维护能力 | V1 发布候选版 |

## 角色分工

| 角色      | 主要负责                     |
| ------- | ------------------------ |
| 你 / 产品  | 定页面范围、定页面目标、定结果页输出、验推荐结果 |
| 内容 / 数据 | 补院校库、路径库、案例、来源、更新时间、风险提示 |
| 前端      | 页面壳、移动端样式、交互、假数据联动、接口接入  |
| 后端      | 数据模型、接口、推荐逻辑执行、结果结构返回    |

---

## 当前最优先的 4 件事

| 优先级 | 事项 | 负责人 |
|--------|------|--------|
| P1 | 把 V1 页面分成必做 / 可延后 / 暂不做 | 你 / 产品 |
| P1 | 确定 V1 首批路径和院校项目范围 | 你 + 内容 / 数据 |
| P1 | 补 10-20 条核心推荐规则 | 你 + 内容 / 数据 |
| P1 | 前端开始做页面壳，后端开始做模型和接口骨架 | 前端 + 后端 |

---

## 建议工作方式

| 节奏 | 建议 |
|------|------|
| 每周 | 以周为单位推进 1 个阶段 |
| 每次同步 | 只对 3 件事：页面范围、数据状态、联调问题 |
| 每次验收 | 按页面验，不按部门验 |
| 每次改动 | 优先改结果页相关内容，再改扩展内容 |
