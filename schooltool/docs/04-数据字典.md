## 一、文档目的

本文件不讨论页面长什么样，也不讨论推荐结论本身是否合理。

本文件只解决一件事：

**把系统里会出现的核心对象和字段先统一。**

只有字段统一，后续这三件事才能并行推进：

- 前端知道每个页面渲染什么
- 后端知道接口和数据表要提供什么
- 内容侧知道每条信息该补到哪个字段里

| 字段 | 类型 | 必填 | 说明 |

| id | string | 是 | 答案唯一标识 |
| question_id | string | 是 | 所属题目 |

**v0.3 跳过题语义补充**：

- `value: null` 在 `/api/answer` 请求中表示"用户主动跳过该题"
- 推荐引擎遇到 `null` 时，按"川渝全样本中位数"补默认值（如 Q5 年龄缺省按 31-35）
- 推荐结果 `recommendation_result.skip_notice` 字段会列出哪些题被跳 + 补值方案
- Q1/Q2/Q3 跳过时前端应阻断提交（`question.is_skippable=false`）

## 3.4 路径方案 `path_option`

用于表达 V1 业务边界内的路径级对象。**V1 路径库 = 2 条主路径**（ww 2026-05-19 裁决封板 · 见 05 §二）：

- `川渝党校`（路径 A · 党校自办在职研究生 · 学信网不可查）
- `非全管综`（路径 B · 教育部全国硕士统考 MPA·MBA·MEM · 学信网可查 · 含川渝党校 MPA 2 个院校项目）

**不入路径库的状态**（由规则层 / 结果页承接，不作 path_option 实体）：

- 大专路径分叉 → 06 B001 规则承接
- R13 英数短板反向追问 → 06 应答框架 dialog_protocol 承接
- 暂不建议立即报考 → 06 B002 结果状态承接

**不在 V1 业务范围**（不入路径库）：合作办学 / 同等学力申硕 / 全日制研究生。

**研知道永久不做该业务**（不入路径库 · ww 2026-05-19 钦定）：教育硕士 / 法学非全 / 先升本 — 研知道**永久不做**该业务，不入路径库，不作独立路径，也不在咨询师建议范围内。

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| id | string | 是 | 路径唯一标识（V1 实例：`path_dangxiao` / `path_feiquan_guanzong`）|
| name | string | 是 | 路径名称（V1 实例：川渝党校 / 非全管综 MPA·MBA·MEM）|
| category | enum | 是 | `川渝党校` / `非全管综`（v0.5 简化：删除"学历过渡 / 暂缓决策"——前者不作独立路径，后者是结果状态由 B002 承接）|
| summary | text | 是 | 路径一句话说明 |
| suitable_for | text | 是 | 适合人群 |
| main_benefit | text | 是 | 核心收益 |
| main_risk | text | 否 | 主要风险 |
| time_cost | string | 否 | 时间成本 |
| money_cost | string | 否 | 费用区间 |
| difficulty_level | enum | 否 | 低 / 中 / 高 |
| recognition_scope | text | 否 | 认可范围说明 |
| status | enum | 是 | 启用 / 备用 / 暂停 |

## 3.13 同画像同学 `peer_profile`（v0.3 新增 · 嵌入 recommendation_result）

PRD v3.2 §4.18「同样画像的同学怎么选」模块的数据契约 · K-NN 5 维相似度 + N≥3 隐私门槛。

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| profile_signature | string | 是 | 用户画像 5 维哈希（用于缓存命中）|
| match_dimensions | json | 是 | 实际匹配的维度列表，如 `["age_band", "tizhi_level", "edu_origin", "region", "value_orientation"]`；至少匹配 4 维才算同画像 |
| matched_count | number | 是 | 同画像样本数 N |
| distribution | json | 是 | 各路径分布数组：每项含 `path` / `count` / `reasons[]`，如 `[{"path":"P001","count":9,"reasons":["不考英数","本系统晋升"]}]` |
| below_threshold | boolean | 是 | `true` = N < 3 隐私门槛不达标，前端展示"样本不足，建议直接咨询顾问 1 对 1 诊断"；`false` = N ≥ 3 正常展示分布 |

**5 维定义**：
- `age_band` 年龄段（与 user_profile.age_range 对齐）
- `tizhi_level` 体制内层级（科员/科级/处级/未明确）
- `edu_origin` 学历起点（与 user_profile.education_level 对齐）
- `region` 地区（与 user_profile.region 对齐）
- `value_orientation` 价值取向（防御型/进取型/防御兼进取）

**数据源**：`cases.json` v0.5.1/V6 正式库（125 条精选脱敏案例）+ M8 学员案例库底账样本派生。

**当前 cases V6 关键字段（v0.5.1 · 29 公开字段）**：
- v0.3 基线（22 字段）：`system_chip` / `education` / `edu_modifier` / `age_concrete` / `work_years` / `unit_narrative` / `is_party_member` / `chosen_path_label` / `narrative_choose` / `reflection` / `story_summary` / `reason_keywords` / `reason_tags` / `story_tags` / `completeness_score` / `display_priority`
- **v0.4 新增 4 字段（2026-05-29 subagent 审计后 ww 钦定）**：`study_method`（方法栈 · #14 详情页"她怎么学的"段）/ `turning_point`（转折点 · #14"决策瞬间"段）/ `engaged_products`（研知道方法引用 list · #14"她用了哪些产品"段）/ `prep_duration`（总耗时段 · #22 chip 备考时长筛选）

字段详见 `cases-json-schema-v0.3-spec.md`（文件名留 v0.3 路径 · 内部版本 v0.5.1）。

**隐私规则（ER3）**：所有个人信息脱敏到地级市 + 单位子类，禁止展示真名 / 单位全称 / 具体岗位。
