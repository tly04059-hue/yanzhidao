# 研知道小程序 — 完整功能与页面结构规划

## Context

研知道要做一个微信小程序，围绕"川渝体制内在职读研"的服务入口。三大模块：首页（服务入口）、择校（核心工具）、备考（备考入口）。

**核心设计原则：**
- 面向非全日制本科学历的体制内人群，注意力和理解力有限
- 能少就少，页面干扰最小化
- 1-2步操作必须非常简单清晰，不让用户动脑
- 3-4步的页面可以信息稍多（用户已有明确目的）
- 用户从打开到获取关键信息/做出判断，控制在3步以内

**操作深度控制：**

| 步数 | 原则 | 示例 |
|------|------|------|
| 第1步 | 一眼看懂，一个动作 | 首页→点"选校诊断" |
| 第2步 | 选项极少，选了就走 | 选系统（六宫格） |
| 第3步 | 可以有信息量，辅助判断 | 结果页（推荐+案例+理由） |
| 第4步 | 行动收口 | 留资/分享/联系顾问 |

### 择校 → 院校匹配建议（诊断流程）

**用户路径：择校Tab → 点卡片 → 六宫格选系统 → 6题 → 结果**

步数分析：
| 步 | 页面 | 复杂度 | 说明 |
|----|------|--------|------|
| 第1步 | 择校Tab点卡片 | 极简 | 1个动作 |
| 第2步 | 答题（六宫格+6题） | 简单 | 每题选一个选项，选了自动下一题 |
| 第3步 | 结果页 | 信息丰富 | 推荐方案+理由+案例+对比+过考率。用户在这里获取信息做判断 |
| 第4步 | 行动 | 收口 | 留资/分享/重测 |

**答题页设计（第2步，必须极简）：**
- 一题一屏，选项大、字大
- 选了**自动跳下一题**，不要"下一步"按钮
- 进度条让用户知道还剩几题
- 条件提示用温和语气，不造成恐慌
- 隐藏底部Tab（沉浸式，不分心）

**结果页设计（第3步，可以信息丰富）：**
7个区块自上而下，但有**视觉层级**——重要的大，次要的小：

1. **诊断头部**（小，一行摘要）
2. **首选推荐**（大卡片，视觉焦点）
3. **备选方案**（小卡片，次要）
4. **相似案例**（中，增加信任）
5. **同行对比**（小，进度条可视化）
6. **信任徽章**（小，过考率+怎么算的）
7. **行动区**（大按钮，明确CTA）

**结果页底部固定栏：**
```
┌──────────────────────────────────┐
│  [重新测试]  [分享同事]  [联系顾问] │
└──────────────────────────────────┘
```

### 择校 → 川渝院校查询

**用户路径：择校Tab → 点卡片 → 院校列表 → 点院校 → 详情**

| 步 | 页面 | 复杂度 |
|----|------|--------|
| 第1步 | 择校Tab点卡片 | 极简 |
| 第2步 | 院校列表（筛选+卡片） | 简单。筛选标签一排，点了就过滤 |
| 第3步 | 院校详情 | 信息丰富。学费/学制/专业/过考率/招生要求 |

**院校列表页：**

- 每个卡片只显示：校名 + 类型 + 地区 + 学费

**院校详情页：**

- 过考率
- 招生要求
- 底部CTA：咨询顾问

## 七、技术方案

| 项 | 选择 | 理由 |
|----|------|------|
| 框架 | uni-app | H5+小程序双端，开发时H5预览 |
| UI库 | Tailwind CSS（自定义） | 完全控制视觉，不要模板味 |
| 后端 | Node.js (Express) | 已有阿里云服务器 |
| 数据库 | MySQL 或 SQLite | 院校数据+用户答题+案例+线索 |

### 开发预览工作流

```
写uni-app代码 → npm run dev:h5 → localhost预览
→ Chrome查看 → 调整 → 热更新
→ 确认后 npm run build:mp-weixin 编译小程序
```
