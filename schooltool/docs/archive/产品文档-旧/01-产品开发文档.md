# 选校工具 / 学历路径决策产品 · 产品开发文档

## 1. 文档目的

本文件用于定义该产品的开发方向、阶段节奏、当前边界与验证方式，作为后续产品需求文档、原型设计和技术实现的总控文档。

## 2. 当前产品定位

这个产品不应仅被理解为“选四川党校的工具”，而应被定义为：

**面向川渝体制内用户的学历路径与职业发展决策产品。**

它解决的不是单一选校问题，而是以下连续问题：

1. 我要不要读研。
2. 我为什么要读研。
3. 我适合哪一条研究生路径。
4. 这条路径的难度、费用、时间和回报是什么。
5. 下一步我应该做什么。

“选校”只是其中一个环节，不是全部。

## 3. 目标用户

当前优先服务川渝体制内有学历提升意愿、但对路径判断存在信息差的人群。

### 3.1 核心人群

- 非全日制本科背景，核心诉求是低成本、相对容易地拿到研究生学历
- 全日制本科背景，希望研究生学历增强单位内晋升、涨工资、竞争上岗的竞争力
- 有研究生诉求，但因英语、数学或全国统考难度过高，转而关注党校路径的人群
- 对学历作用只有模糊认知，希望通过工具理解学历与职业发展的关系的人群

### 3.2 当前不优先覆盖的人群

- 非川渝地区用户
- 以学术深造、科研或全日制脱产升学为核心诉求的人群
- 只关注全国统招院校排名、不关心体制内适配逻辑的人群

## 4. 核心问题

用户表面上是在问“考四川党校还是别的”，本质上是在解决以下问题：

- 自己为什么需要研究生学历
- 研究生学历在体制内到底有什么作用
- 不同学历路径之间如何比较
- 哪条路径更适合自己当前的年龄、岗位、学历起点和预算
- 如何减少信息差带来的决策焦虑

当前咨询成本高的根源，不是信息太少，而是信息分散、判断标准不统一、解释重复发生。

## 5. 产品目标

### 5.1 用户价值

- 帮用户快速判断自己适合哪条读研路径
- 帮用户理解学历与职业发展的关系
- 帮用户坚定“我要不要考研”这件事
- 帮用户形成下一步行动，而不是只得到一堆信息

### 5.2 业务价值

- 降低重复咨询成本
- 形成高信任、低门槛的引流产品
- 通过公开、透明、系统化的信息建立品牌信任
- 为后续顾问承接、课程转化、长期产品化积累基础结构

## 6. 产品形态判断

小程序、App、H5 都只是呈现形式，不是当前第一优先级。

当前优先级应是：

1. 把产品结构、入口、页面关系和关键结果想清楚
2. 做成可交互、可验证、可调整的前端体验原型
3. 再决定以什么技术形态落地

因此当前阶段的重点不是后台和数据，而是：

- 信息架构
- 页面关系
- 路径判断逻辑
- 结果表达结构
- UI 风格方向

## 7. 开发阶段规划

### Phase 1：产品定义与结构验证

目标：

- 明确产品定位、用户分层、核心价值
- 明确第一版到底做“路径判断”还是“完整选校”
- 形成开发文档、PRD 和专项设计文档

交付物：

- 产品开发文档
- V1 产品需求文档
- 路径诊断与结果结构设计文档

### Phase 2：交互原型与页面体验验证

目标：

- 把核心链路做成可点击、可走完一遍的体验原型
- 验证页面顺序、内容密度、结果结构、CTA 收口是否合理

交付物：

- 首页
- 路径诊断页
- 结果页
- 院校比较页
- 备考承接页

### Phase 3：内容体系与规则沉淀

目标：

- 把职业发展解释、学历作用、典型人物案例、院校路径差异沉淀成内容模块
- 把推荐规则、提示规则、风险提示标准化

交付物：

- 路径对比内容库
- 用户案例库
- 结果页内容模板
- 推荐规则与提示规则说明

### Phase 4：技术与数据实现

目标：

- 根据已经稳定的页面和逻辑，再接后台、数据库、配置系统
- 支持内容维护、规则更新、用户记录与数据统计

交付物：

- 数据结构设计
- 后台录入方案
- 推荐逻辑实现
- 小程序 / App 技术落地方案

## 8. 当前版本边界

当前阶段先不做：

- 复杂后台系统
- 完整数据管理平台
- 过度精细的推荐算法
- 全国范围扩展
- 高复杂度社交、社区或学习管理功能

当前阶段先做：

- 产品定义
- 页面结构
- 路径判断逻辑
- 可点击体验版本

## 9. 当前建议的开发顺序

1. 先完成产品开发文档
2. 再明确 V1 产品需求文档
3. 再定义路径诊断和结果结构专项文档
4. 然后做原型 / 前端 Demo
5. 原型稳定后再进入数据与技术实现

## 10. 验证标准

第一阶段判断是否有效，不看技术完成度，主要看以下几项：

- 用户是否能快速理解这个产品是干什么的
- 用户是否能顺利走完一条完整路径
- 结果页是否能让用户觉得“有价值”
- 工具是否能替代一部分重复咨询解释
- 是否能有效支持留资、咨询或进一步转化

## 11. 当前结论

当前项目应从“选校工具”升级理解为“体制内学历路径决策产品”。

第一版最重要的，不是做全，而是先把这件事做对：

**帮助用户看清自己适合哪条路径，并说明为什么。**
