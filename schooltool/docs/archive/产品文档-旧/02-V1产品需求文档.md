# 选校工具 / 学历路径决策产品 · V1 产品需求文档

## 1. 版本目标

V1 的目标不是做一个“全功能平台”，而是做一个能够帮助用户完成第一轮判断的轻量产品：

**帮助川渝体制内用户快速判断自己适合哪一条读研路径，并给出清晰的下一步动作。**

## 2. V1 要解决的问题

V1 重点解决以下三件事：

1. 帮用户明确“我为什么要读研”
2. 帮用户判断“我应该走哪条路径”
3. 帮用户理解“下一步该做什么”

V1 不追求一次性解决全部问题，而是要先完成“路径判断 + 结果解释 + 初步承接”。

## 3. V1 核心功能

### 3.1 首页

作用：

- 让用户一眼知道产品在帮他解决什么问题
- 引导进入核心诊断
- 建立基础信任感

首页应传达的核心信息：

- 这不是单纯选四川党校，而是帮你判断适合哪条学历路径
- 你可以通过几分钟诊断获得第一轮答案
- 这是研知道整理出的体制内路径信息，不是杂乱信息堆砌

### 3.2 路径诊断

作用：

- 通过若干关键问题，把用户归入不同路径类型

V1 诊断重点围绕：

- 学历起点
- 地区
- 党员身份
- 年龄阶段
- 当前系统 / 岗位方向
- 核心目标
- 预算范围
- 对考试难度的现实接受度

### 3.3 结果页

作用：

- 输出用户最适合的路径
- 解释为什么
- 给出备选方案
- 给出下一步动作

结果页不是简单推荐学校，而应输出：

- 你的首选路径
- 为什么适合你
- 主要收益
- 主要风险
- 难度 / 时间 / 费用概览
- 备选路径
- 下一步建议

### 3.4 院校 / 路径比较页

作用：

- 让用户在已有初步判断后继续深入理解

V1 比较重点：

- 四川党校 vs 重庆党校
- 党校 vs MPA
- 单证路径 vs 双证路径
- 低成本路径 vs 更高回报路径

### 3.5 备考与顾问承接页

作用：

- 把资料、备考、咨询等收口动作统一起来

V1 先不做复杂学习系统，只做：

- 资料领取
- 备考时间线
- 顾问联系入口

## 4. V1 页面关系

建议的主链路如下：

1. 首页
2. 为什么读研 / 简短引导区块
3. 路径诊断
4. 结果页
5. 深入比较页
6. 咨询 / 资料领取

也可以保留更轻版本：

1. 首页
2. 路径诊断
3. 结果页
4. 咨询 / 资料领取

## 5. V1 的用户路径

### 路径 A：模糊意向用户

- 打开首页
- 看到“这是帮你判断路径的工具”
- 进入诊断
- 得到首选路径和解释
- 再决定是否继续咨询

### 路径 B：已有党校意向用户

- 带着“四川党校”意向进入
- 通过诊断发现自己未必适合党校，或适合重庆党校 / MPA
- 在结果页理解差异
- 进入顾问承接

### 路径 C：已有学历焦虑但目标不明的用户

- 通过首页和结果页理解学历对职业发展的作用
- 被推动从模糊焦虑进入清晰判断

## 6. V1 的核心输出结果

V1 结果页最重要的输出，不是“推荐某校”，而是“推荐某条路径”。

建议的首选结果类型包括：

- 四川党校路径
- 重庆党校路径
- MPA / 教育硕士路径
- MBA / 管理类路径
- 先补本科，再进入研究生路径
- 当前暂不建议立即报考，先准备

## 7. V1 的业务目标

- 降低初级咨询重复成本
- 形成可传播、可体验的低门槛引流产品
- 提升用户对研知道的信任感
- 为顾问承接提供更清晰的用户前置信息

## 8. V1 当前不做的内容

- 复杂账号体系
- 强依赖后台的数据维护系统
- 完整学习管理
- 复杂分享裂变机制
- 全国范围院校扩展

## 9. V1 的成功标准

- 用户能在短时间内看懂产品价值
- 用户能完成诊断并理解结果
- 用户觉得结果页“有帮助，不空”
- 工具能明显减少顾问前期解释成本
- 工具能承接咨询和资料领取

## 10. V1 当前建议

V1 最适合先做成：

**“路径诊断 + 结果解释 + 初步承接”的轻量决策产品。**

不要一开始就把它做成沉重的数据库平台或复杂内容平台。
