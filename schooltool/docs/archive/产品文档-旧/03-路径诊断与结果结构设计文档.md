# 选校工具 / 学历路径决策产品 · 路径诊断与结果结构设计文档

## 1. 文档目的

本文件用于定义该产品最核心的能力：

- 如何对用户进行分层
- 如何通过诊断问题完成路径判断
- 结果页应该输出什么结构

## 2. 用户分层模型

当前建议先按“底层诉求”而不是按学校来分。

### 2.1 学历补短板型

特征：

- 多为非全日制本科背景
- 核心诉求是尽快提升学历层级
- 更关注成本、难度、时间和可实现性

适合输出重点：

- 哪条路径更容易实现
- 哪条路径性价比更高
- 是否适合先低成本拿到研究生学历

### 2.2 单位竞争力增强型

特征：

- 多为全日制本科背景
- 核心诉求是增强晋升、涨工资、竞争上岗的能力

适合输出重点：

- 学历在单位竞争中的实际作用
- 不同路径对单位内发展的差异
- 费用和回报的平衡

### 2.3 考试难度规避型

特征：

- 本质上想读研究生
- 但因为英语、数学、全国统考难度高，转向党校

适合输出重点：

- 为什么党校成为可行替代
- 党校与双证路径的核心差别
- 这种选择的收益与边界

### 2.4 模糊焦虑型

特征：

- 只有“我是不是也该有研究生”的模糊念头
- 还不清楚学历能带来什么

适合输出重点：

- 为什么读研
- 学历对职业发展可能产生的作用
- 是否值得现在开始考虑

## 3. V1 诊断目标

V1 诊断的目标不是做精确算法，而是完成第一轮路径归类。

也就是说，诊断后的结果应是：

- 你更适合走哪条路径
- 为什么
- 如果不走这条路径，备选是什么

## 4. V1 诊断问题框架

建议诊断围绕以下问题展开：

1. 你所在的地区
2. 你的学历起点
3. 你的年龄阶段
4. 你所在的系统 / 岗位
5. 你的党员身份
6. 你读研最核心的目标
7. 你对考试难度的接受度
8. 你的预算范围

这些问题的作用不是简单收集信息，而是完成以下判断：

- 你是“补学历”还是“增强竞争力”
- 你是否更适合党校
- 你是否更适合双证路径
- 你是被预算约束，还是被考试难度约束
- 你当前是否适合立即开始

## 5. 路径判断框架

当前建议先定义 6 类结果：

1. 四川党校主路径
2. 重庆党校主路径
3. MPA / 教育硕士主路径
4. MBA / 管理类主路径
5. 先补本科再进入研究生路径
6. 暂不建议立即报考，先准备

## 6. 结果页结构

结果页建议分为以下区块：

### 6.1 结论区

直接告诉用户：

- 你的首选路径是什么
- 你属于哪类人群

### 6.2 原因区

解释：

- 为什么这条路径适合你
- 你当前的关键约束是什么
- 这条路径对应解决什么问题

### 6.3 成本与难度区

展示：

- 时间成本
- 费用区间
- 考试难度
- 进入门槛

### 6.4 价值区

解释：

- 这条路径对你的职业竞争力意味着什么
- 适合的使用场景是什么
- 不能解决什么问题

### 6.5 备选区

给出：

- 如果不走首选路径，第二选择是什么
- 为什么是备选而不是首选

### 6.6 相似案例区

作用：

- 帮用户代入
- 帮用户坚定决策

案例方向建议包括：

- 类似年龄
- 类似岗位
- 类似学历起点
- 类似目标

### 6.7 下一步动作区

输出具体行动，而不是抽象建议。

例如：

- 去看哪类院校
- 领取资料
- 咨询顾问
- 先补本科
- 再做深度诊断

## 7. 内容表达原则

### 7.1 不只回答“选什么”，还要回答“为什么”

用户真正缺的不是学校名单，而是判断框架。

### 7.2 不只给结论，还要给解释

只有结论，没有解释，用户不会信。

### 7.3 不只给路径，还要给下一步动作

否则结果页只是“看完即走”。

## 8. V1 专项建议

V1 结果页应优先突出以下 4 件事：

1. 路径判断
2. 路径原因
3. 成本 / 难度 / 回报
4. 下一步动作

人物案例、职业发展解释、历史人物参考等，可以作为 V1.5 或 V2 强化模块继续补充。

## 9. 当前结论

这个产品最核心的能力，不是“列学校”，而是：

**把模糊的学历焦虑，转化为清晰的路径判断。**

因此专项设计的重点，不是院校数据库本身，而是诊断逻辑和结果结构。
