# 研知道·选校诊断小程序 — 产品方案

> 版本：v1.0 | 日期：2026-03-11
> 用途：交给 Gemini/开发者 Build 的完整产品方案
> 前置文档：design-2026-03-06-选校诊断工具设计.md（推荐引擎逻辑）

---

## 一、产品概述

### 1.1 一句话定义

**川渝体制内人的在职读研选校诊断工具**——回答6-8道题，3分钟得到个性化选校方案 + 真人案例 + 顾问对接。

### 1.2 为什么是小程序（而非H5）

| 能力 | H5 | 小程序 | 对研知道的价值 |
|------|-----|-------|-------------|
| 微信分享卡片 | 链接（被折叠） | 原生卡片（有封面+标题） | 体制内人在工作群里分享，卡片点击率远高于链接 |
| 服务通知 | 无 | 可推送1次 | 用户做完诊断后，可推"你的完整报告已生成" |
| 订阅消息 | 无 | 用户授权后可定期推 | 招生季推"今年XX校招简更新了" |
| 微信授权登录 | 需额外开发 | 原生支持 | 用户画像更完整 |
| 收藏/最近使用 | 困难 | 自动出现在历史列表 | 留存路径 |
| 打开速度 | 取决于网络 | 有缓存，秒开 | 体制内人网络环境不稳定（尤其民族地区） |

### 1.3 核心价值

- **对用户**：3分钟搞清楚自己适合读什么，看到和自己一样的人怎么选的
- **对研知道**：每个访客自动生成完整画像（系统/地区/目标/预算/年龄/学历/党员），顾问拿到的不是微信号，而是一份诊断卡
- **对品牌**：过考率公开透明 → 行业唯一 → 自然信任

## 二、页面结构 & 用户流程

### 2.1 页面清单（共5页）

```
首页（身份入口页）
  ↓
答题页（6-8题，一题一屏）
  ↓
加载过渡页（2-3秒，"正在匹配..."）
  ↓
结果页（诊断报告）
  ↓
分享/留资页（生成海报 or 填微信号）
```

### 2.2 详细页面设计

**参考对标：薄荷健康小程序首页 + 蜗牛保险"智能核保"入口页**

薄荷健康的做法：首页不是功能列表，而是一个大按钮"测一测你的健康状况"，下面是"已有XXX人完成测评"。蜗牛保险更进一步——首页就是身份分流："我是XX岁的XX，想了解XX保险"。

**设计要点：**

1. **六宫格是第一道题**——选了系统之后直接进入答题流，不需要额外的"开始测试"按钮。点击即开始，减少一步操作
2. 每个格子用**图标+系统名+补充说明**（如"组织/宣传"），帮用户快速定位
3. 底部两行是**信任锚点**，不是按钮，不可点击，纯展示
4. 整体色调：白色背景 + 深蓝色文字 + 红色/金色点缀。不要用绿色（支付），不要用黑色（丧）
5. LOGO区域简洁，不放slogan，不放多余文案

**首页的人群分流机制：**

用户选了不同系统后，后续答题的问题会动态变化：

| 首页选了 | 答题流程的差异 |
|---------|-------------|
| 党政机关 | Q岗位选项：组织部/宣传部/纪委监委/财政局/发改委/... |
| 公检法纪检 | Q岗位选项：公安/法院/检察院/纪委/... + 法学推荐权重提高 |
| 乡镇街道 | 若年龄25-30 → 追加5年服务期提醒 |
| 教育医疗 | Q岗位选项新增：管理岗/专技岗 → 专技岗强制MPA |
| 国企银行 | Q目标选项新增"转公务员"；MBA权重提高 |
| 民族地区 | 自动标记民族地区专项名额（150人） |

**所以"人群分流"不是单独的页面功能，而是首页选择直接决定了后续的整条答题路径。**

#### 页面2：答题页（6-8题，一题一屏）

**参考对标：丁香医生"症状自查" + 网易云音乐"年度报告"的答题交互**

丁香医生的答题体验：每页一题，选了自动滑到下一题，顶部有进度条，可以回退修改。网易云的年度报告加了一层——每题之间有微动效过渡，选项点击有触感反馈。

**答题页结构：**

```

│  ████████░░░░░░░░░░░░  28%           │
│                                      │
│                                      │
│     你是中共党员吗？                   │
│     （含预备党员）                     │
│                                      │
│                                      │
│     ┌──────────────────────────┐     │
│     │                          │     │
│     │   ○  是（含预备党员）       │     │
│     │                          │     │
│     └──────────────────────────┘     │
│                                      │
│     ┌──────────────────────────┐     │
│     │                          │     │
│     │   ○  不是                 │     │
│     │                          │     │
│     └──────────────────────────┘     │
│                                      │
│                                      │
│                                      │
│                                      │
│                                      │
└──────────────────────────────────────┘
```

**题目顺序（首页已回答系统，所以从第2题开始）：**

| 序号 | 题目 | 选项 | 逻辑作用 |
|------|------|------|---------|
| 首页 | 你在哪个系统工作？ | 六宫格 | 决定后续岗位选项、专业匹配、院校推荐 |
| Q2 | 你是中共党员吗？ | 是/否 | 非党员+四川→排除四川党校 |
| Q3 | 你在四川还是重庆工作？ | 四川/重庆 | 决定院校范围 |
| Q4 | 你的最高学历是？ | 全日制本科/非全日制本科/大专 | 大专硬阻断；学历类型影响话术 |
| Q5 | 你的年龄段？ | 25-30/31-35/36-40/41+ | 遴选年龄提醒、案例匹配 |
| Q6 | 你读研的核心目标是什么？ | 动态（根据系统） | **核心分流**：党校 vs MPA |
| Q7 | 你的具体岗位方向？ | 动态（根据系统） | 专业方向精确匹配 |
| Q8 | 你的预算范围？ | 2-3万/5-8万/8万+ | 预算兜底交换 |

**交互要点：**

1. **选了自动跳下一题**，不需要"下一步"按钮。点击选项 → 0.3秒延迟（让用户看到选中状态）→ 自动滑动到下一题
2. **可以回退**：顶部"← 返回"或左滑回到上一题
3. **进度条**：顶部细线进度条，带百分比
4. **条件触发提示**：当某个选择触发排除规则时，在选项下方滑出提示卡片（不阻断流程）

**条件提示卡片样式：**

```
┌──────────────────────────────────────┐
│  ⚠️ 温馨提示                          │
│                                      │
│  四川党校要求中共党员身份。             │
│  你可以报考重庆党校（不要求党员）        │
│  或选择 MPA 院校。                     │
│                                      │
│  系统已自动调整推荐方案。               │
│                            [ 知道了 ] │
└──────────────────────────────────────┘
```

这个卡片参考**微信支付成功后的提示条**——从底部滑入，不遮挡页面主体，点"知道了"收起，不打断答题节奏。

### 2.3 信任徽章（详细设计）

**参考对标：大众点评"黑珍珠餐厅"认证标 + 支付宝"芝麻信用分"展示 + 丁香医生底部"审核医师"标识**

信任徽章不是一个独立页面，也不是看板，而是**结果页中间的一个静态信息条**，大约占屏幕1/4高度。

**具体样式：**

```

│  │ · 包含所有报名学员               │   │
│  │                                │   │
│  │ 数据周期：每年录取结束后更新       │   │
│  │ 最近更新：2025年8月              │   │
│  └────────────────────────────────┘   │
│                                        │
└────────────────────────────────────────┘
```

**设计要点：**

1. **数字要大**——40.4% 和 50% 是视觉焦点，用 36px+ 加粗字体
2. **"行业唯一"四个字**——不是吹牛，是事实。全行业没有第二家公开真实过考率的
3. **"怎么算的？"展开区域**——默认折叠，点击展开。这块是核心壁垒：你不是数字高，而是**你敢说你怎么算的**。行业其他机构宣称85-95%但不公布计算方式
4. **对比基准**——"全国管综自然过线率约25%"，让用户自己感受差异
5. **不需要图表、不需要趋势线、不需要动效**——就是两个数字+一段计算说明。简单、真实、可验证

**视觉参考：**

- 数字展示参考**芝麻信用分**的大数字居中样式
- "怎么算的？"折叠面板参考**微信支付"账单详情"**的展开方式
- 整体容器参考**大众点评黑珍珠**的认证卡片——浅色底+细边框+小图标

**一年更新一次的处理方式：**

- 数据标注"2025级"年份，用户自然理解
- 每年7-8月录取结果出来后，后台改两个数字即可
- 不需要"看板"那种实时刷新的形态

### 3.1 决策流程

```
用户答题完成
    ↓
第一层：目标分流
    目标=遴选/转公/职称 → MPA优先
    目标=本单位晋升/防御 → 党校优先
    ↓

    非党员+四川 → 排除四川党校

### 8.1 MBA大师估分小程序 — 专项参考分析

> MBA大师（薛睿创办，2015年）：累计220万用户，92%自然流量，免费课获客→付费私教变现。
> 估分小程序是其核心获客工具之一，每年管综考试当天上线。

**MBA大师估分小程序的产品流程：**

```
选择考试科目（管综/英语二）
  ↓
逐题输入答案（选择题直接选，主观题填估分）
  ↓
提交 → 出估分结果
  ↓
看全国/地区排名分布（"你超过了XX%的考生"）
  ↓
CTA：加入备考群 / 查看课程 / 分享对答案
```

**值得研知道参考的6个设计点：**

#### ① 时间窗口获客

MBA大师估分小程序不是常年可用的——考完试当天才爆发，但窗口内获客效率极高。

研知道的等价时间窗口：

| 窗口期 | 事件 | 运营动作 |
|--------|------|---------|
| 3-4月 | 党校招生简章发布 | 推"今年招简变了，重新测一下" |
| 5-6月 | 管综提前批/报名季 | 推"还没决定报哪所？测一下" |
| 9-10月 | 考研正式报名 | 推"报名倒计时，最后确认方案" |
| 12月考后 | 管综笔试结束 | 可扩展"估分+调剂方案"功能（直接学MBA大师） |

**落地建议**：小程序常年可用，但在窗口期配合公众号推文/短视频集中推广。每个窗口期可以更新首页Banner或弹窗提示最新变化。

#### ② 输入门槛极低

MBA大师不问"你是谁"——上来就选答案。到出结果那一步才触发留资。

**对研知道的确认**：首页六宫格"选系统"即第一道题，不要注册、不要填信息、点了就开始。这个设计正确，保持。

#### ③ 结果页的社交货币属性

估分结果页让用户想截图发考研群——"我估了XX分，你呢？"天然社交货币。

**对研知道的落地**：
- 分享海报不要出现"培训机构"字样，用"川渝体制内选校诊断"中性标题
- 匹配度数字（如95%）有分享冲动——"我匹配度比你高"
- 海报调性要"正经"，体制内人分享到工作群不能觉得low

#### ④ 同行对比机制（重要）

MBA大师告诉你"你的分数超过了XX%的考生"——排名制造对比感和紧迫感。

**对研知道的落地**：选校没有高低，不能做排名。但可以做"同类人选择分布"：

```
┌────────────────────────────────────┐
│  👥 和你相似的学员怎么选的           │
│                                    │
│  在与你情况相似的 186 位学员中：      │
│                                    │
│  ████████████████░░░░  78% 选了党校  │
│  ████░░░░░░░░░░░░░░░░  22% 选了MPA  │
│                                    │
│  其中已上岸的学员过考率：             │
│  党校 40.4% ｜ 管综 50%             │
│                                    │
└────────────────────────────────────┘
```

这个功能基于1123人真实数据，可以按系统×学历类型×目标做交叉统计。既是"同行者"内容的产品化，又自然嵌入了过考率信任徽章。

**建议加在结果页的"相似案例"区块之前或之后。**

#### ⑤ 结果分步加载

MBA大师的结果不是一次全出——先出总分，再展开各科，再出排名。每步有短暂加载感。

**对研知道的落地**：
- 加载过渡页（"分析系统背景 ✓ → 匹配院校 ✓ → 查找案例..."）已做
- 结果页本身也可以分步：首选方案先出来，往下滑才看到备选→案例→同行对比→信任徽章→CTA
- **不要一次性全展开**，让用户有"越往下看信息越丰富"的层次感

#### ⑥ 工具型入口 → 内容型留存

MBA大师的小程序是工具（估分），用完引导进入内容生态（免费课/社群/薛睿账号）。工具一次性，内容持续。

**对研知道的落地**：选校诊断也是一次性工具。在结果页底部CTA区域增加一个轻入口：

```

## 九、注意事项（给开发者）

1. **推荐逻辑是核心资产**，全部放后端，前端只传答案、接收结果JSON
2. **案例是真实学员脱敏数据**，不能编造，所有案例由ww提供和审核
3. **过考率数据一年更新一次**，后台可改，不需要实时计算
4. **所有院校数据需要ww确认**后才能上线，招生简章每年变
5. **移动端优先**，95%用户从微信打开，PC端不需要适配
6. **字要大、按钮要大**，主要用户35岁+
7. **不做注册登录**，用微信openid静默获取即可
8. **分享卡片的标题和封面**很重要——体制内人在工作群分享，卡片要"正经"，不能花哨
