# 川渝体制内在职读研选校诊断工具 — 项目设计

> 版本：v1.1 | 日期：2026-03-10（逻辑更新：人群洞察对齐）
> 定位：研知道引流+顾问辅助的互动H5工具
> 部署：阿里云服务器

---

## 一、产品定义

### 1.1 一句话描述

用户回答6-8道选择题，系统基于研知道真实业务逻辑，给出个性化的选校推荐+理由+类似案例，底部引导加顾问微信获取深度诊断。

### 1.2 使用场景

| 场景 | 入口 | 用户行为 |
|------|------|---------|
| 引流 | 公众号推文/朋友圈/短视频评论区 | 点链接→答题→看结果→加微信 |
| 顾问辅助 | 顾问发链接给客户 | 客户填完→顾问收到画像→精准开聊 |

### 1.3 核心价值

- **对用户**：3分钟搞清楚自己适合读什么，不用盲目问
- **对研知道**：每个访客自动生成画像（系统/地区/目标/预算/年龄/学历/党员），顾问拿到的不是一个微信号，而是一份完整诊断卡

```
    |  教育医疗 → 教师/医护/学校医院党务岗/综合岗

    |
    v
[问题5] 你读研的核心目标是什么？
    |  - 在本单位稳步晋升（职级并行/竞争上岗）
    |  - 想遴选到上级机关
    |  - 不确定，但不想在学历上被比下去
    |  - 职称评审需要（事业编专用选项，根据问题3动态显示）
    |  - 想从国企/事业编转公务员（根据问题3动态显示）
    |
    v
[问题6] 你的年龄段？
    |  - 25-30岁
    |  - 31-35岁
    |  - 36-40岁
    |  - 41岁以上
    |
    v
[问题7] 你的最高学历是？
    |  - 全日制本科
    |  - 非全日制本科（自考/成教/函授/网教/国开）
    |  - 大专
    |
    v
[问题8] 你的预算范围？
    |  - 2-3万（性价比优先）
    |  - 5-8万（可以投入）
    |  - 8万以上（不是主要考虑因素）
    |
    v
[生成结果页]
```

### 2.1 动态逻辑说明

- **问题4根据问题3联动**：不同系统展示不同的具体岗位选项
- **问题5根据问题3联动**：事业编显示"职称评审"选项，国企显示"转公务员"选项
- **非党员+四川**：在问题2后插入提示，不阻断流程，但影响后续推荐（排除四川党校）
- **大专学历**：四川党校不接受单大专（需双大专），提示并调整推荐

### 3.3 专业匹配规则

#### 四川党校（3个专业，12个方向）

| 用户岗位/系统 | 推荐专业 | 推荐方向 | 逻辑 |
|-------------|---------|---------|------|
| 组织部/宣传部/统战部/机关党委 | 政治学 | 党建 | 党的队伍和组织体系建设 |
| **公安系统** | **政治学** | **社会治理** | 警衔+行政双轨都认党校，政治学招生多更稳妥。备选法学 |
| **法院/检察院** | **法学** | **法治社会** | 法检系统对口度最高。未入额走行政序列可选政治学 |
| 纪委监委 | 政治学 | 社会治理 | 也可选法学（党内法规），看个人背景 |
| 应急管理局/应急相关 | 政治学 | 应急管理 | 直接对口 |
| 民政局/人社局/街道办/社区 | 政治学 | 公共管理 | 公共事务管理对口 |
| 想成为干部培训对象（不限岗位） | 政治学 | 党建/科社 | 覆盖面最广，招生最多 |
| 财政局/经信局/发改委/统计局 | 经济学 | 经济学/区域经济 | 财经系统对口 |
| 涉及经济金融岗位 | 经济学 | 经济学 | 不考数学，政治经济学 |
| 法治/普法/信访调解/制度规范 | 法学 | 法治社会 | 有法律背景优先 |
| 党内法规/纪检相关 | 法学 | 党内法规 | 需要法学基础 |
| 无明确对口+无专业背景 | 政治学 | 公共管理 | 默认推荐：招生最多、覆盖最广、入门门槛最低 |

**专业选择提示规则**：
- 法学招生仅100人，过考率6.9%（最低）→ 无基础不推荐，系统自动提示
- 经济学不是西方经济学，不考数学 → 系统自动说明
- 政治学招300人，过考率10.9%（最高）→ 作为"安全牌"推荐

#### 重庆党校（5个专业）

| 用户岗位/系统 | 推荐专业 | 逻辑 |
|-------------|---------|------|
| 组织部/宣传部/纪委监委/统战部/国企党委办 | 党政管理 | 覆盖面最大，党的队伍和领导体系 |
| **公安系统** | **公共管理** | 警衔+行政双轨，公共管理覆盖面最广 |
| **法院/检察院** | **法律** | 法检系统对口 |
| 民政局/人社局/应急/街道办/社区 | 公共管理 | 最广泛的拓展性 |
| 发改委/政研室/外事办/大型国企战略部 | 战略管理 | 领导能力与前瞻布局 |
| 财政/经济/金融相关 | 经济管理 | 有经济基础或直接工作需要 |
| 法律/法治/合规相关 | 法律 | 有法学背景优先 |
| 无明确对口+无专业背景 | 公共管理 | 默认推荐：适合所有体制内人员 |
| 党政vs公管拿不准 | 两个都展示+差异说明 | 党政=管人，公管=管事 |

**重庆专业选择提示规则**：
- 公管和党政参考书最薄 → 备考资料量最小
- 法律和经济专业性强 → 无基础日常接触少的人入门门槛高
- 各专业过考率12%-14.7%，差异不大 → 不要纠结这点差异

#### MPA/MBA院校匹配

| 用户系统+地区 | 推荐院校 | 逻辑 |
|-------------|---------|------|
| 四川省厅/成都市直 | 四川大学MPA | 省市两级校友网络最强 |
| 财税/金融系统 | 西南财经大学MPA | 系统校友统治力 |
| 交通/住建/水利 | 西南交通大学MPA | 行业对口+2年制 |
| 重庆市直/区县 | 重庆大学MPA | 重庆系统最高认可 |
| 教育系统 | 四川师范/西南大学 | 系统内校友密度远超985 |
| 国企管理岗 | MBA（川大/电子科大/重大） | 企业系统MBA认可度高于MPA |

### 3.4 排除/提示规则（共9条，v1.1更新）

| # | 条件 | 动作 |
|---|------|------|
| 1 | 非党员+四川 | 排除四川党校，推荐重庆党校（明确告知四川人可报重庆） |
| 2 | 大专学历 | 硬阻断→直接跳结果页引导先提升本科 |
| 3 | 目标=遴选 | 提示：遴选不认党校学历，必须双证MPA |
| 4 | 教育医疗+教师/医护 | 提示：党校学历在职称评审中一般不认，双证可缩短2-3年 |
| 5 | **重庆+遴选+36岁以上** | 提示：重庆市级遴选上限35岁（2025年公告） |
| 5b | **四川+遴选+41岁以上** | 提示：四川省直遴选上限40岁，中央遴选科员38/副科43/处级46 |
| 6 | 目标=转公务员 | 提示：调任需副处级/副高+组织推荐，MPA双证是基础件 |
| 7 | 法学推荐时 | 提示：法学招生仅100人，过考率6.9%最低 |
| **8** | **乡镇街道+25-30岁** | 提示：5年服务期内不能调动/遴选，读研是唯一能做的晋升准备 |
| **9** | **事业编+专技岗** | 提示：职称不认党校学历，强制MPA优先 |

### 3.5 推荐理由话术系统（v1.1新增）

根据用户学历类型自动切换话术风格：

| 学历类型 | 党校推荐前缀 | 党校推荐后缀 | MPA推荐前缀 |
|---------|------------|------------|------------|
| 全日制本科 | "让你的优势更明显。" | "巩固现有条件，在组织考察和竞争上岗中更有底气。" | "双证MPA让你的学历优势扩展到更多场景。" |
| 非全日制本科 | "不让学历成为你唯一的短板。" | "档案里多一笔实打实的研究生学历，在职级并行和竞争上岗中不被比下去。" | "双证硕士是从非全日制本科到研究生学历的质变。" |

**逻辑来源**：人群洞察报告第三章 — 52%全日制本科为"巩固型"，48%非全日制本科为"补短板型"。

## 四、案例库设计

### 4.1 案例来源

1. **13份学员访谈**（01-人群洞察/咨询访谈/25学生访谈实例.docx）— 有详细动机、学历、单位、学习历程
2. **1123人学员数据** — 提炼典型画像
3. **上学记反馈** — 上岸后真实体验

### 4.2 案例结构

每条案例包含：

```
{
  "id": "case_001",
  "name": "杨姐",              // 脱敏化名
  "age": 35,
  "gender": "女",
  "city": "甘孜",
  "system": "医疗系统",
  "position": "县医院后勤岗",
  "education": "成教本科",
  "party_member": true,
  "work_years": 10,
  "goal": "本单位晋升",
  "choice": "四川省委党校·政治学",
  "motivation": "周围同事都是博士，只有极少数是本科，感受到了学历差距",
  "quote": "拥有研究生学历能为我打开更多职业通道",
  "outcome": "备考中/已上岸/在读",
  "tags": ["医疗系统", "非全日制本科", "补短板型", "民族地区"]
}
```

### 4.3 案例匹配算法

按权重匹配用户画像与案例库：

| 匹配维度 | 权重 | 说明 |
|---------|------|------|
| 系统相同 | 30% | 同系统的人最有共鸣 |
| 目标相同 | 25% | 留/走/防御的动机一致 |
| 学历类型相同 | 20% | 全日制vs非全日制，决策逻辑不同 |
| 地区相近 | 15% | 同城市/同区域 |
| 年龄相近 | 10% | ±5岁范围 |

取匹配度最高的案例展示。如无高匹配案例，展示同系统的通用案例。

### 4.4 初期案例数量

V1版本准备**20-25条**典型案例，覆盖：

- 6个系统 × 2种学历类型 = 12条基础覆盖
- 民族地区3条（甘孜/阿坝/凉山）
- 遴选转向型2条
- 延退觉醒型（38-42岁）2条
- 国企留后路型2条
- 上学记体验型2条（增加信任感）

## 五、信息收集与防护

### 5.1 渐进式信息收集

| 阶段 | 收集内容 | 用户感知 |
|------|---------|---------|
| 打开页面 | 访客ID（UUID）、来源渠道、设备信息、时间戳 | 无感 |
| 每道题作答 | 实时存储每步选择，关联访客ID | 无感（用户以为在做测试） |
| 结果页 | 推荐结果、停留时长、是否截图 | 无感 |
| 点击"获取完整报告" | 微信号/手机号，关联前面所有数据 | 主动提交 |

**顾问看到的数据**：
```
[新线索] 2026-03-06 14:32
微信号：wxid_xxx
画像：成都 / 财政局 / 党员 / 全日制本科 / 32岁 / 想遴选 / 预算8万+
推荐结果：西南财经大学MPA（首选）/ 省委党校经济学（备选）
匹配案例：王哥，33岁成都税务局
来源：公众号推文《两会后川渝体制内该怎么接招》
```

### 5.2 防爬与数据保护

| 防护层 | 措施 |
|-------|------|
| **前端防护** | 推荐逻辑不放前端，所有计算走后端API |
| **API防护** | 接口签名+时间戳校验，防止直接调用API |
| **频率限制** | 同一IP/设备，10分钟内最多完成3次测试 |
| **内容保护** | 案例文本通过API动态加载，不写死在HTML中 |
| **结果页防护** | 结果页核心内容用Canvas渲染（截图可以，但爬虫抓不到文本） |
| **推荐逻辑保护** | 决策树和权重参数存在后端配置，不暴露给前端 |
| **来源追踪** | 每个分享链接带唯一渠道参数，知道流量从哪来 |

### 5.3 信息互检真伪机制（后续迭代）

| 机制 | 说明 |
|------|------|
| 学历验证 | 用户填写学历后，可选上传学信网截图，后台人工核实 |
| 单位验证 | 顾问加上微信后，通过朋友圈/对话确认单位信息真实性 |
| 交叉验证 | 同一微信号多次测试，系统对比前后答案一致性 |
| 异常标记 | 短时间内大量不同画像的测试 → 标记为可疑（可能是竞品） |

## 六、技术方案

### 6.1 架构

```
[用户手机/微信内置浏览器]
         |
    [Nginx - 阿里云]
         |
    [前端 H5] ←→ [后端 API]
                      |
                [数据库 MySQL/SQLite]
                      |
            [管理后台（顾问端）]
```

### 6.2 技术选型

| 层 | 选型 | 理由 |
|----|------|------|
| 前端 | Vue3 + Vant（移动端UI库） | 轻量、移动端体验好、微信内H5兼容 |
| 后端 | Node.js (Express/Koa) 或 Python (FastAPI) | 你的阿里云服务器已有，快速开发 |
| 数据库 | MySQL（阿里云已有的话）或 SQLite（轻量启动） | 存用户答题数据+案例库+推荐日志 |
| 部署 | 阿里云ECS + Nginx | 已有服务器，零额外成本 |
| 域名 | 挂在研知道现有域名下（如 tool.yanzhidao.com） | 品牌一致 |

### 6.3 数据表设计

```sql
-- 访客记录（每次访问一条）
visitors:
  id, uuid, source_channel, device, ip, created_at

-- 答题记录（每道题一条）
answers:
  id, visitor_id, question_key, answer_value, created_at

-- 推荐结果
recommendations:
  id, visitor_id, primary_school, primary_major, alternative_school,
  case_id, fit_score, created_at

-- 线索（用户主动留信息）
leads:
  id, visitor_id, wechat_id, phone, created_at

-- 案例库
cases:
  id, name, age, gender, city, system, position, education,
  party_member, goal, choice, motivation, quote, outcome, tags

  id, name, type(党校/MPA/MBA), province, majors, fee,
```

## 九、数据准确性保障

### 9.1 院校数据校验清单

| 数据项 | 来源 | 校验方式 |
|-------|------|---------|
| 四川党校招生简章 | scge.gov.cn官网 + 研知道推文 | ww人工确认 |
| 重庆党校招生简章 | cqdx.gov.cn官网 + 研知道推文 | ww人工确认 |
| MPA院校学费 | 各校官网招生简章 | 每年更新 |
| 过考率数据 | 研知道真实统计 | ww提供原始数据 |
| 报考条件 | 官方招生简章 | 每年招简发布后更新 |

### 9.2 逻辑校验清单（防低级错误）

| 校验项 | 预期 |
|-------|------|
| 非党员+四川 → 不推荐四川党校 | 必须通过 |
| 目标=遴选 → 不推荐党校作为首选 | 必须通过 |
| 事业编教师/医护 → 提示党校对职称无用 | 必须通过 |
| 法学无背景 → 提示风险 | 必须通过 |
| 大专学历 → 提示四川党校不接受 | 必须通过 |
| 民族地区 → 展示专项名额信息 | 必须通过 |
| 四川经济学 → 说明不考数学 | 必须通过 |
| 每个系统×每个目标组合都有推荐结果 | 必须通过 |
| **重庆+遴选+36岁以上 → 提醒35岁上限** | 必须通过 |
| **四川+遴选+41岁以上 → 提醒40岁上限** | 必须通过 |
| **乡镇+25-30岁 → 提醒5年服务期** | 必须通过 |
| **事业编+专技岗 → 强制MPA优先+职称提醒** | 必须通过 |
| **公安系统 → 推荐政治学（社会治理）** | 必须通过 |
| **法院检察院 → 推荐法学（法治社会）** | 必须通过 |
| **非全日制本科 → 推荐理由含"不让学历成为短板"** | 必须通过 |
| **全日制本科 → 推荐理由含"让优势更明显"** | 必须通过 |

### 9.3 更新机制

- 每年3-4月招生简章发布后，更新院校数据
- 每年7-8月录取结束后，更新过考率数据
- 案例库持续补充（每批新学员访谈后）
- 所有数据更新需ww确认后才上线

## 十、关键风险与应对

| 风险 | 应对 |
|------|------|
| 推荐逻辑出错导致用户投诉 | Phase 1上线前全量测试所有组合；结果页底部标注"仅供参考，具体请咨询顾问" |
| 竞品抄袭 | 核心逻辑后端不暴露；案例动态加载；持续更新内容形成壁垒 |
| 数据过时 | 建立年度更新流程，招简发布后第一时间更新 |
| 用户做完不留信息 | 结果页价值足够但留有"完整报告"钩子；分享传播本身也是价值 |
| 微信封链接 | 域名备案合规；不做诱导分享；备用域名预案 |
