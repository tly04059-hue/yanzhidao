## 三、Q5 年龄段：3 套口径 ⚠️

**这是最常踩坑的口径**。3 套段不同。

| 来源 | 段定义 | 段数 |
|---|---|---|
| `questions.json` Q5 options value | 25-30 / 31-35 / 36-40 / 41+ | 4 段 |
| `strategy_pool.json` match.age_bands | 25-30 / 30-35 / 35-40 / 40-45 / 40+ | 5 段（按种子约定）|
| `cases.json` age_band 字段 | 25-30 岁段 / 30-35 岁段 / 35-40 岁段 / 25-35 岁段（民族州 10 岁段）/ 40-45 岁段 / 45+ 岁段 / 30-40 岁段 | **7 段 + "岁段" 后缀** |

### 映射规则

**Q5 → strategy.age_bands**（strategy_pool _meta.value_mapping.Q5_to_age_band_seed）：

```jsonc
{
  "25-30": "25-30",
  "31-35": "30-35",
  "36-40": "35-40",
  "41+": "40+"
}
```

**Q5 → cases.age_band**（matchCases 引擎需要做）：

```js
const Q5_TO_CASE_AGE = {
  '25-30': { exact: ['25-30 岁段'], partial: ['25-35 岁段'] },  // 民族州 10 岁段半重叠
  '31-35': { exact: ['30-35 岁段'], partial: ['25-35 岁段','30-40 岁段'] },
  '36-40': { exact: ['35-40 岁段'], partial: ['30-40 岁段','40-45 岁段'] },
  '41+':   { exact: ['40-45 岁段','45+ 岁段'], partial: [] }
};
```

**派生脚本（derive_cases_json.py）派生时 → cases.age_band**：

```python
def age_to_band(age):
    a = int(age)
    if a < 25: return None  # 跳过
    if a <= 30: return "25-30 岁段"
    if a <= 35: return "30-35 岁段"
    if a <= 40: return "35-40 岁段"
    if a <= 45: return "40-45 岁段"
    return "45+ 岁段"
```

## 十、case_id：1 套口径（v0.3/V6 正式库）

格式：`C{序号:03d}`

- 当前正式库：`C001-C126`，共 125 条（历史上 `C007` 跳过；投稿与 v04 合并后存在多来源连续接力）
- `case_id` 只保证稳定引用，不表达来源批次；来源批次看 `key_quote_source` / `_m8_case_id` / `meta.v04_merge_status`

## 十一、trigger_user_segments fuzzy matching（v1.1 新增）

`policy_basis.json` 的 cards / additional_policy_signals 字段 `trigger_user_segments` 是**自由文本**（如 "36-40 岁" / "国企/央企在职" / "目标=遴选"），需要 fuzzy matching 映射到 9 题答案。

详细规则在 `policy_basis.json` `_meta._trigger_user_segments_standardization_v01_2026_05_27` 段。

### 速查（5 类 segment）

| Segment 模式 | 映射规则 |
|---|---|
| "36-40 岁" / "41 岁以上" / "32-38 中层" / "25-30 岁" | Q5 年龄段比较 |
| "国企/央企在职" / "教师" / "医疗" / "党政机关" / "公检法" / "乡镇基层" / "事业编" | Q1 系统比较（bank_finance 含国企双 tag）|
| "目标=遴选" / "目标=转公务员" / "目标=职称" / "目标=晋升" / "目标=防御" | Q6 目标比较 |
| "党员" / "非党员" | Q2 党员比较 |
| "民族州" / "重庆" / "四川" | Q3 地区比较 |
| **"所有用户"** | **always true**（兜底命中所有人） |

不在上述模式内的 segment 默认不命中（不漏报但需后续补规则）。
