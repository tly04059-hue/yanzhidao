## 一、产品定位

### 这个产品是什么

**为川渝体制人在职考研提供学历路径决策服务。**

不是"帮你选四川党校的工具"，而是帮用户完成从"我要不要读研"→"我为什么要读"→"我该走哪条路"→"下一步做什么"的完整判断链条。

"选校"只是其中一个环节，不是全部。

### 战略身份（在经营战略中是什么）

本产品在《经营战略总纲 §3.1 三层飞轮》中是**产品层**：

| 维度            | 战略定位                                               | 出处                       |
| ------------- | -------------------------------------------------- | ------------------------ |
| 飞轮分层          | 三层嵌套飞轮的**产品层**（主飞轮经营层 + 上下文库能力层 + 小程序产品层）          | 总纲 §3.1.2                |
| 产品类型          | **工具型辅助产品**（vs 党校在职 / MPA 备考课程是换取型核心产品）            | 商业全景 §4.4.1              |
| 主飞轮作用         | 节点 ①→③ 的工具载体（择校判断 + 决策推导 + 留微信）                    | 总纲 §3.1.2                |
| 上下文库作用        | **上下文库事实层的前端入口**（用户侧数据直接入库）                        | 总纲 §3.1.2                |
| 小程序自身飞轮       | 积累越多 → 匹配越准 → 用户越多 → 数据越多 → 回到起点                   | 总纲 §3.1.2                |
| 立场            | **防御性产品**（应对结构性焦虑、保留职场可能性、提高下限）                    | 业务必读 §5.1                |
| AI 时代叙事       | **第五股力量 · 确定性投资动机**——AI 免疫的、可确定的成就                 | 商业全景 §2.1 + 总纲 §2.3.1    |
| 起点承诺三层        | 结果层：降低决策成本 + 提升服务体验 + 提高备考效率                       | 业务必读 §一 + 总纲 §3.1.2 节点 ① |
| 触点 tagline 短版 | "不骗人真服务，稳定提高过考率"（与公众号 / 短视频 / 咨询 / 推荐共用）           | 业务必读 §一 对外品牌承诺           |
| 时序            | Year 1 Phase 2 上线 → Year 1 末用户数据可测 → Year 2 进入复利窗口 | 总纲 §3.1.3 / §六           |
| 窗口期           | **1-2 年内产品化完成**（窗口过期飞轮转不起来）                        | 总纲 §5.5 壁垒边界             |
| 北极星指标         | **扫码添加微信率**（不是答题率、不是注册量）                           | 总纲 §六 Phase 2            |
| 内容获客位置        | 自媒体内容（决策阶段信息需求）→ 小程序工具（择校诊断 + 院校匹配）→ 微信            | 总纲 §4.2                  |

**这意味着 3 件事**：
1. 产品功能取舍以"是否服务于上述战略身份"为唯一裁决标准
2. 凡是与上述任一身份冲突的功能（进攻性话术 / 强引导分享 / 隐私前置询问 / 去人化对话）一律不做
3. V1 不追求规模指标，追求**留微信率** + **数据可入上下文库的完整度**

### 产品的三层能力

| 层   | 能力                  | 承载形态           |
| --- | ------------------- | -------------- |
| 解释层 | 公开信息结构化，讲清学历作用和路径差异 | "了解"Tab（浏览式）   |
| 匹配层 | 根据个人画像推荐具体方案        | "测一测"Tab（答题式）  |
| 承接层 | 导流到顾问、引入备考服务        | 首页入口 + 结果页 CTA |

### 一句话定义

川渝体制内人的在职读研路径决策工具 — 公开信息的结构化整理和政策解读、按你情况匹配的真实案例、3 分钟给出个性化选校方案。

### 产品形态

微信小程序和H5。原因：

- 微信分享卡片原生支持，体制内人工作群分享点击率远高于链接
- 服务通知和订阅消息可推送
- 微信授权登录、自动出现在历史列表
- 体制内人网络环境不稳定（尤其民族地区），小程序有缓存秒开
- H5页面方便不习惯使用微信或者在微信生态以外流转

## 二、目标用户

### 核心人群（按读研意向阶段分）

| 阶段  | 用户心理           | 产品接入方式                    |
| --- | -------------- | ------------------------- |
| 犹豫期 | "我是不是也该读个研？"   | "了解"Tab：浏览政策、案例，理解学历作用和案例 |
| 决策期 | "我该考党校还是 MPA？" | "测一测"Tab：答题得出个性化方案        |
| 行动期 | "我已经决定了，怎么备考？" | 首页入口 → 备考方案、资料、联系顾问       |

**三个阶段不是强制顺序，产品三个入口并列存在，用户可以从任何阶段进入。**

### 核心人群（按画像分）

| 类型         | 特征             | 核心诉求              |
| ---------- | -------------- | ----------------- |
| 补短板型（~48%） | 非全日制本科，28-40 岁 | 低成本拿到研究生学历，不被学历卡住 |
| 巩固型（~52%）  | 全日制本科，26-40 岁  | 增强晋升竞争力，让优势更明显    |
| 难度规避型      | 想读研但怕统考        | 党校作为可行替代，了解差异     |
| 模糊焦虑型      | "是不是也该读个研"     | 需要政策解释和案例帮他理清     |

### 当前不覆盖

- 非川渝地区用户
- 以学术深造、全日制脱产为目标的人
- 只关注全国院校排名、不关心体制内适配逻辑的人

## 三、核心价值

### 对用户

- 模糊期：看政策 + 看案例，理解学历到底有什么作用（不是鸡汤，是政策依据）
- 决策期：3 分钟搞清楚自己适合读什么，看到和自己一样的人怎么选的
- 行动期：得到清晰的下一步动作 — 备考方案 / 资料 / 顾问

### 对研知道

- 每个访客自动生成完整画像（系统/地区/目标/预算/年龄/学历/党员）
- 顾问拿到的不是一个微信号，而是一份判断书
- 降低重复咨询成本，把重复解释变成标准化产品
- 复合型引流产品：低门槛进入，浏览式留存 + 答题式转化
- 品牌信任：过考率公开透明 + 政策d依据明确 + 不卖课卖判断

### 核心差异

市面上没有同类产品。现有咨询成本高的根源不是信息太少，而是信息分散、判断标准不统一、解释重复发生。这个产品把研知道的研究资产（政策研究 + 人事制度 + 院校数据 + 学员案例）公开透明地结构化，用户能自己看明白，不只靠顾问解释。

## 四、开发阶段规划

### Phase 1：产品定义与结构验证（当前阶段）

**目标：** 想清楚、画清楚、能体验

- 收敛产品定位文档
- 确定页面结构和流程
- 做出可交互原型
- 确定视觉风格

**交付物：**
- 产品开发文档（本文件）
- V1 PRD
- 交互原型（HTML）

**验证标准：**
- 用户能快速理解产品是干什么的
- 能顺利走完一条完整路径
- 结果页让用户觉得"有价值"

**不做：** 后端、数据库、管理后台

### Phase 2：内容与规则沉淀

**目标：** 确保数据准确、逻辑正确

- 院校数据逐项审核
- 案例库正式消费层就绪（当前 `cases.json` 125 条精选脱敏案例）
- 推荐规则全组合验证
- 所有文案审核

**交付物：**
- 院校数据集（JSON）
- 案例库（JSON）
- 题目配置（JSON）
- 内容审核通过记录

**验证标准：**
- 所有院校数据 ww 确认
- 逻辑校验清单全部通过
- 案例可公开、无隐私风险

### Phase 3：技术实现

**目标：** 把产品做出来

- 技术方案定稿
- 前端开发（小程序所有页面）
- 后端开发（推荐引擎 API）
- 前后端联调

**交付物：**
- 可运行的小程序
- 后端 API 服务
- 数据库

### Phase 4：测试与上线

**目标：** 确认正确、部署发布

- 全流程端到端测试
- 真机 iOS/Android 测试
- 部署到阿里云
- 微信小程序提审发布

### Phase 5：运营与迭代

**目标：** 推广、监测、优化

- 渠道推广（公众号、朋友圈、顾问 1v1）
- 数据监控（漏斗、转化率、画像分布）
- 持续优化（A/B 测试、案例补充、话术调整）

## 五、当前版本边界

### V1 三 Tab 结构

```
[ 首页 ]   [ 了解 ]   [ 测一测 ]
```

### V1 先做（与 PRD v3.3 对齐 · 形态级清单）

**首页（总入口）**
- 三个主动作入口：了解 / 测一测 / 备考
- 信任数据锚：四川党校 40.4% · 重庆党校 44% · 管综 55.6% → 87.5%（初试 → 复试）· 1,000+ 川渝同学服务样本
- 首页 tagline 短版："不骗人真服务，稳定提高过考率"（与其他触点共用）
- 其他入口：院校查询 / 联系顾问 / 使用说明 / 数据来源 / 关于研知道

**"了解"Tab（长文全景图 · v3.3 形态升级）**
- **取消四宫格 + 4 个浏览详情页旧形态**
- 一份完整的「川渝体制内学历路径全景」长文（6 段线性铺开）
- 用户**不需要做任何选择**（不选维度 / 不答题 / 不填信息），往下滑到底就能拿走完整认知
- 6 段：政策大背景 / 学历到底有什么用 / 不同情况下规则不一样（4 维度横扫）/ 4 条路径总览 / 同行实际怎么选 / 你可以做的事

**"测一测"Tab（答题式 + 互通通道）**
- 6-8 道题（命中 DP001 时扩到 9-11 步）
- **每题加「先去了解」入口**（一键跳到了解 Tab 对应章节锚点 + 保留答题进度）
- **每题加「跳过本题」**（结果页提示"基于你跳过的 X 题，Y 部分是川渝平均"）
- 7 决策维度（含 Q9 多选顾虑题：能力偏好 + 客户主因）
- DP001 英数短板反向追问（inline 三层追问）
- 加载过渡页 + 个性化结果页（区块 1 含 skip_notice）

**结果页（v3.2 商业飞轮 4 项补强）**
- 7 区块结构：首选 + 备选 + 路径意义 + 案例 + 信任数据 + CTA + 分享好物中心
- **4 类分享物**（A 结果卡 / B 政策卡 / C 故事卡 / D 判断书图片）替代原"分享海报"单一物
- **判断书做成图片**（不做 PDF），不带 ww 个人署名，结尾含风险与边界文案
- **改答案重新测**（不清空 + 对比新旧结果）
- **同样画像的同学怎么选**（K-NN 5 维 + N≥3 隐私门槛）

**备考内容（从首页入口进，不做一级 Tab）**
- 党校 / 管综两层决策树（管综再分 MBA / MPA / MEM 专业选）
- 免费资料领取 / 加入备考群 / 课程服务介绍

**API 收敛**
- 4 个 `/api/browse/*` 收敛为 1 个 `/api/understand`（v3.3）

**其他**
- 留资功能（结果页 + 独立资料领取入口）
- 院校筛选直接显示 [四川][重庆][云南][贵州]
- share_click 埋点加 method 维度（self / friend / trusted / poster）

### V1 不做

- 复杂账号体系和登录
- 数据管理后台（Phase 2 再做）
- 精细的推荐算法（决策树 + 元规则 + DP 应答框架足够）
- 全国范围扩展（V1 只覆盖川渝）
- 社区 / 学习管理功能
- 分享裂变机制（先做基础分享 + 不强引导）
- 独立的备考 Tab（备考从首页进）
- "了解 Tab 四宫格 + 4 个浏览详情页"（v3.3 已废弃）
- 任何在了解 Tab 上要求用户付出信息的设计（v3.3 形态硬约束）
- 任何"强制走完才给结果"的设计（必须有跳过 + 逃生通道）

## 八、核心原则

1. **先想清楚，再动手写代码。** Phase 1 不碰技术实现。
2. **用户体验 > 功能完整。** 3 步内完成任何目标。
3. **数据准确 > 数据丰富。** 宁可少做院校，不能错。
4. **推荐逻辑是核心资产。** 全部在后端，前端不碰。
5. **所有院校数据和案例需 ww 确认后才能上线。**
6. **字要大、按钮要大。** 主要用户 35 岁+。
7. **调性要正经。** 体制内人分享到工作群不能觉得 low。
