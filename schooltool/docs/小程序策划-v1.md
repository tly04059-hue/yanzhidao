## 一、产品定位（3 句话）

**1. 一句话定位**

选校诊断小程序是研知道**学历决策全链路的入口型产品**——把"要不要读研、走哪条路、下一步做什么"压缩为一份 3 分钟可拿到的判断书，面向川渝体制内 30-40 岁人群。

**2. 在研知道业务版图里的角色**

依据：经营战略总纲 §3.1.2 三层嵌套飞轮 · 小程序飞轮。

- **前端引流**（主）：抖音/小红书/公众号/学员推荐均可承接，沉淀到一个具备判断输出能力的产品页，而非空白企微号
- **决策辅助**（核心价值）：消化 M1-M9 上下文库的判断能力，把咨询师隐性规则（M7 §七 12 条规则 + R13 应答框架）显化成可触达的诊断逻辑
- **私域承接**（出口）：把"想留资 / 想咨询 / 想看方案"分流到加企微，**留资的是画像不是空号**
- **数据反哺**（飞轮）：每次答题、每次跳出、每次企微对话，都回流成上下文库的样本（M5 触发场景、M7 隐性规则、M8 案例）

**3. 与竞品差异化**

依据：业务必读 §5.1-§5.5、价值观层 §3.3 信任方程式。

行业竞品（中公、华图、品睿、蓉曜云等）的小程序多是"课程货架 + 留资引流"——表达层是销售推送。研知道的差异点是**"判断书"而不是"课程清单"**：不卖课、不催单、不要求公开传播；提供研知道的判断、不替你做决策；公开真实过考率而不是模糊承诺。**对应信任公式：分母（自我导向感）极低 + 分子（可信度/可靠度/亲密感）通过数据透明 + 顾问承接拉高。**

## 二、目标用户（3 个核心分层）

依据：业务必读 §一目标客群（30-40 岁体制相关在职人群）、M1 §一-§五、M5 §五客户底层框架 + §七 S1-S7 触发场景、M3-学历作用 §二客户画像与学历需求对应关系。

按 V1 产品消费场景把 7 个触发场景归并到 3 个核心分层（S6/S7 主要由销售 TMDR 承接，不是小程序首战场）：

### 分层一 · 防御占位型（核心客群，对应 S1 + S3）

- **画像**：32-38 岁 / 公务员或事业编 / 本科 / 多数党员 / 县乡或市直机关
- **触发场景**：35 岁红线焦虑（M5 §S1） + 单位推动（M5 §S3 软推荐）
- **没有研知道时怎么决策**：在中公/品睿对比留资被反复电话推销 → 凭朋友圈机构话术做决策 → 错配到不对路的项目（如英数恐惧者被推党校但党校自然过考率 < 10%，依据 M9 §4.1·注 + M7 §R6）
- **痛点**：信息密度高但筛选成本高、不敢被同事看到在朋友圈做"考研选择题"（M5 §5.2 圈子重合约束）
- **小程序如何解决**：首页 4 卡真实过考率打底（可信度）→ "了解" Tab 给完整地图（无控件、无压力）→ "测一测"出判断书（亲密感）→ 私域承接（避免公域转发）

### 分层二 · 路径模糊型（高增长客群，对应 S2 + 部分 S4）

- **画像**：30-36 岁 / 国企中层或事业编专技岗 / 本科或后取本科 / 党员身份不一 / 主城区为主
- **触发场景**：同侪刺激（M5 §S2，同事/同学/配偶刺激） + 裸考失败后想"看看怎么选"（M5 §S4 部分）
- **没有研知道时怎么决策**：被同侪情绪带 → 直接报与同侪同款（忽略个体差异）→ 选错路径或选错院校
- **痛点**：动机来自外部刺激而非自身规划，对"5 种路径有什么区别"几乎没有结构化认知；对"党校 vs MPA"二选一有概念但说不清差异（M3-五种路径 §一对比表是空缺认知）
- **小程序如何解决**：长文 §3 4 维度横扫帮其定位 + §4 4 路径对比表（按系统/目标/学历/年龄+党校 vs MPA vs MBA vs MEM 11 维度）补足结构化认知；测一测 Q9 触发 DP001 反向追问，破除"英数恐惧 → 党校"的简化误判

### 分层三 · 资格刚需型（高单价客群，对应 S3 硬要求 + 部分 S1）

- **画像**：32-42 岁 / 公务员（想遴选/调任）或事业编专技岗（教师/医护要职称） / 本科或硕士进修第二专业 / 党员居多
- **触发场景**：单位硬要求双证（M5 §S3 B 分支） + 35 岁红线下的最后一程冲刺（M5 §S1 A 分支）
- **没有研知道时怎么决策**：知道要双证，但在 71 个川渝 MPA 项目 + 67 个 MBA 项目中没法做学校层面的精细选择（M9 §3）
- **痛点**：高决策压力但低试错容忍度——读错院校 3 年时间 + 5-22 万学费打水漂
- **小程序如何解决**：测一测命中遴选/职称/调任 → 触发 R4 元规则（必须双证）→ 院校库按 R10 元规则不打"主力客群"标签 + 按系统对口给 3 个候选 + 跳企微做 1v1 院校精细化判断

> ⚠️ S6（机构对比）/ S7（1v1 临门一脚）不在 V1 主战场——这两个场景的应答路径在销售 TMDR 而非小程序内。小程序的作用是把这些客户筛进私域，由销售承接。

## 三、产品价值主张（按信任公式拆解）

依据：上下文价值观层 §3.3 信任方程式 = (可信度 × 可靠度 × 亲密感) ÷ 自我导向感、§3.4 各环节说服策略、§3.7 AI 时代立场。

研知道的优势在**分母（自我导向感）极低**——"机构不是在想怎么赚我的钱"。小程序四个变量必须同时设计。

### 可信度 ↑：研知道凭什么让用户相信

- **首页 4 卡真实过考率**：四川党校 40.4% / 重庆党校 44%（25 年） / 管综 2026 初试 55.6% → 复试 87.5%（含调剂）——目前未见同行做（价值观层 §3.3）
- **数据严格分省/分阶段不合并**：党校川/渝独立、管综初试 → 复试明示递进，违反 ER2 元规则的笼统过考率不出现（PRD §三 ER2）
- **§4.20 过考率说明页**："怎么算的 / 历年数据 / 招生量背景 / 数据来源" 全暴露——让用户能验证、能反推
- **政策原文文号**：所有引用必须有原文号（中央〔2000〕10 号 / 国企改办发〔2021〕7 号 + 国务院国资委 2024-09-27 会议 / 2026 国考公告等，依据 M3-学历作用 §六）

### 可靠度 ↑：研知道凭什么让用户安心

- **判断书是"研知道判断"不是"AI 通用模板"**：所有规则有 ≥3 处证据来源（M7 §七.1 元规则）
- **元规则 ER1 不打"主力客群"标签**：不说"你属于研知道最擅长服务的群体"——这是基于经营战略总纲 §5.2 公理第 ⑥ 类的诚实表达
- **元规则 ER2 用"可能性和判断方向"口径**：禁用"一定会 / 必然 / 保证"等绝对句式（价值观层 §3.8 C2）
- **同行实际怎么选**（长文 §5）：1,000+ 全样本 + 王哥案例 → 用真实分布而非话术说服

### 亲密感 ↑：研知道凭什么让用户感觉"懂我"

- **答题 8 题 + Q9 多选顾虑**：用户主诉"英语不行"不直接推党校，先跑 R13 三层反向追问（M7 §7.4 应答框架）—— **听懂用户的话再回话**
- **结果页区块 3 "这个路径对你意味着什么"**：把抽象路径名转译成"在你的系统里这意味着什么"——基于 M3-学历作用 §一 7 大场景作用分场景判断
- **§4.6 题旁"先去了解"逃生通道**：用户随时可跳了解 Tab 对应锚点不丢答题进度——尊重"我没准备好答下一题"的真实心理
- **加企微页面承诺**："你问什么答什么 / 你不问的不主动 / 什么时候付费由你定"——把销售反向承诺写进产品

### 自我导向感 ↓：研知道凭什么不让用户觉得"机构有所图"

- **首页不放"立即报名" / "限时优惠"**：CTA 是"先了解 / 先测一测 / 看看时间"，全部是用户视角的功能词（PRD §4.1）
- **结果页区块 8 取消所有分享 CTA**：旧版 6 个分享按钮全砍——分享 = 抬高自我导向感（机构想要用户裂变），违反业务必读 §5.1 圈子重合保护
- **保存判断书是"长图存手机"而非"分享朋友圈"**：让用户拥有判断资产，不让其被同事看到（M5 §5.2 三个硬约束）
- **不做"跳过本题"压力答题**：可跳过 + 跳过项用川渝平均补值 + 明示"哪几题用平均值补的" → 用户感觉自己仍有控制权

## 四、信息架构原则

依据：业务必读 §5.2 客户行为特征、M5 §5.3-§5.4、PRD §二信息架构 + §三设计原则。

### 用户心理状态 → 3 个 Tab 的映射

研知道识别用户有 **3 种主导心理状态**，对应 3 个 Tab，不是漏斗式层层逼近。

| 心理状态                | Tab     | 核心动作                 |
| ------------------- | ------- | -------------------- |
| 犹豫期（"是不是也该读个研？"）    | **了解**  | 浏览长文 / 不做选择 / 拿走规则地图 |
| 决策期（"我该考什么？"）       | **测一测** | 答 8 题 / 拿建议 / 进入私域   |
| 行动期（"我决定了 / 想直接联系"） | **首页**  | 看备考方案 / 加企微 / 看院校库   |

### 跳转原则

- **不做漏斗强压**：3 个 Tab 平行、可来回切换 / 长文每段都有"我准备好了 → 测一测"的弱诱导，没有"必须看完才能测"的卡断
- **给逃生通道**：测一测每题旁"先去了解" → 跳了解 Tab 对应锚点 → 保留进度返回（PRD §4.6）；用户随时可放弃当前流程不被惩罚
- **节奏由用户定**：不弹窗、不催完成、不限时——尊重 M5 §5.3 "周末/晚上集中处理"的特征

### 主路径 vs 辅助层

- **主路径**：首页 → 三选一（先了解 / 先测一测 / 先看备考方案）→ 各 Tab → CTA → 加企微
- **辅助层**：院校库 / 学员地图 / 过考率说明 / 数据来源 / 使用说明 / 关于研知道——挂在首页"其他入口"列表，弱视觉权重，不污染主路径

## 六、关键产品决策的策划依据（10 个）

依据：每个决策附"上下文依据 + 段号"，便于反向追溯。

### D1. 3 个 Tab 而不是 5 个或 1 个

**依据**：业务必读 §5.2 客户行为特征"消费型强、创造型弱" + M5 §5.4 两类心态（投机型/认真备考型）。
**策划逻辑**：3 个 Tab 对应 3 种主导心理状态（犹豫 / 决策 / 行动），少则切不开心理状态，多则给体制内"不犯错"人群增加决策负担。

### D2. 取消"跳过本题"的硬限制，改为允许跳过+平均值补值+明示

**依据**：M5 §5.2 第二硬约束"不犯错导向" + 价值观层 §3.5 不催单红线 + PRD §4.6 + 区块 1 跳过提示。
**策划逻辑**：体制内用户对"被迫表态"非常敏感（不犯错 = 不留把柄）。强制每题必答会触发防御反应直接退出。允许跳过、补值并明示哪几题用了平均值——既保留判断书完整性，又让用户感觉控制权在自己。

### D3. 取消首页"分享好物中心" / 分享 CTA / 海报裂变

**依据**：业务必读 §5.1 防御性产品 + §5.2 圈子重合 + 价值观层 §3.3 自我导向感最大资产、§3.4 各环节说服策略"越上游越不能有销售痕迹"。
**策划逻辑**：分享/裂变本质是"机构想要用户帮忙获客" → 抬高分母（自我导向感）+ 把用户暴露给同事（违反圈子重合保护）。研知道的获客逻辑是"内容获客不做销售推送"，小程序不能违反这条。**取代方案**：保存判断书到手机（私域决策资产）+ 加企微 1v1（亲密感）。

### D4. 元规则 ER1 不打"主力客群"标签

**依据**：经营战略总纲 §5.2 公理第 ⑥ 类（公理化的诚实立场）+ M7 §7.0 R10 + M4 §八。
**策划逻辑**：行业惯例是"你属于我们最擅长的群体"——这是诚实问题。研知道的事实是规则按场景适用，不按"哪类客户最多"概率推荐。展示文案、推荐文案、加企微首句都不能出现"你是我们主力服务的群体"。

### D5. 元规则 ER2 用"可能性和判断方向"口径

**依据**：价值观层 §3.8 C2 + M7 R11 + M9 §4.1·补"基于核实政策事实的可能性"。
**策划逻辑**：AI 时代用户更容易被"必然预测"误导（如"延迟退休后 MPA ROI 必然上升"）。研知道的立场是给"可能性 + 判断方向"，让用户不错过政策信号但不被必然句式锁死。结果页区块 3 第 4 段"为什么这个时候做这件事"严格按此口径写。

### D6. Q9 用"主诉顾虑多选 + 可不选 + 触发反向追问"而非"主诉顾虑必填"

**依据**：M7 §7.4 R13 应答框架（v0.2 校准） + M9 §4.1·注 "英数恐惧是误导性优势" + 价值观层 §五团队判断语言库"理科思维/文科思维/说不准"用户语言级校准。
**策划逻辑**：用户主诉"英语不行"的真实可能性有 4 种（能力真伪 / 动机需求 / 能力偏好 / 努力偏好），直接"英语短板 → 党校"会把双证刚需用户错配到党校（党校自然过考率 < 10% 比双证更难考）。Q9 多选+可不选符合"不催答"，触发后跑三层反向追问让用户先表态再推荐。

### D7. "了解" Tab 用一份长文滚到底而不是树形/Tab/分页结构

**依据**：业务必读 §5.2 内容偏好"务实信息 > 观点输出" + M5 §5.3 行为特征"消费型强、不愿做选择" + PRD §4.2 设计要点"不允许任何选维度控件"。
**策划逻辑**：体制内用户在长文上的耐心高于在多层菜单上的耐心——"不让选"反而是减少决策负担的设计选择。给一份完整地图，用户滚到自己关心的段落停下来即可，符合周末/晚上集中处理的节奏。

### D8. 结果页区块 3 嵌入"这个路径对你意味着什么"而非只给推荐院校

**依据**：M3-学历作用 §一 7 大场景作用 + 经营战略总纲 §0.4 起点承诺的具象层（判断书 / 陪跑地图）+ PRD §4.8 区块 3。
**策划逻辑**：研知道卖的不是"推荐院校"而是"判断书"——推荐院校只是判断书的结论，判断过程（在你的系统里政策怎么作用 / 真实作用数据 / 不能解决什么 / 为什么这个时候做）才是真正的差异化价值。这一段把"了解"Tab 的政策解释能力嵌入到结果页，让用户感觉"研知道是在解释，不是在推销"。

### D9. 信任数据 4 卡放首屏，且严格分省/分阶段不合并

**依据**：价值观层 §3.3 信任方程式可信度抬高路径 + 业务必读 §一对外触点话术（四川党校 40.4% / 重庆党校 44% 严格区分）+ PRD §4.1 设计要点。
**策划逻辑**：首屏是用户对研知道的第一信任锚点。模糊合并的过考率（如"川渝党校平均 42%"）违反 ER2 元规则，也违反价值观层"敢承诺具体数字"的声音特征。4 卡可点跳"过考率说明"——让用户随时可验证。

### D10. V1 不上"账号体系/收藏/历史"

**依据**：M5 §5.2 圈子重合（不留个人痕迹的偏好）+ PRD V1 MVP 范围 + 业务必读 §5.7 能力复用度判断。
**策划逻辑**：小程序登录注册增加决策成本，且体制内用户对"留下身份痕迹"敏感。V1 用 session 内状态（答题进度、判断书）+ 加企微做长期承接，**身份在企微里、不在小程序里**——这与研知道的私域承接逻辑一致。账号体系等 V2 数据反哺成熟后再评估。

## 七、V1 边界与 V1.1 / V2 规划

### V1 边界（已在 PRD 中）

20 个页面 + 2 个组件，主线：首页 / 了解 / 测一测 / 院校库 / 结果页 / 加企微。**不做**：账号体系、付费转化页、备考方案的全流程化（V1 只是简版）、垂直路径（教育硕士 / 法学非全 — **永久不做**，不在产品边界内）。注：MEM 保留，MEM 是路径 B（非全管综）下的学位类型，是 V1 主推路径之一，不是独立垂直路径。

### V1.1 规划（3-6 个月后）

## 八、数据反哺机制

依据：经营战略总纲 §3.4 上下文库飞轮（事实层 / 价值观层 / 流程层 / 案例层）+ M5 §三"100+ 结构化样本"目标 + M7 §三 v1.0 升级条件。

V1 小程序产生 4 类回流数据，各对应一个上下文库消费方：

| 数据类型 | 回流形式 | 上下文库消费方 | 用途 |
|---|---|---|---|
| **答题分布** | Q1-Q8 选项统计 + 跳出节点 + 完成率 | M4 场景矩阵 + M1 人群分类 | 校对"川渝体制内人群构成"是否与既有判断一致 |
| **Q9 主诉顾虑分布** | 7 个顾虑选项的命中频次 + DP001 触发率 + 三层追问通过率 | M5 触发场景 + M7 隐性规则 | 校对 R13 三层追问设计是否有效 / 提炼 W1-W6 中哪几条能升 §七 |
| **企微对话主题** | 加企微后第一句话题分布 / 客户成交后的画像对比 | M5 §七 触发场景清单 + M7 §八 rule-watch | 补 §三目标 100+ 结构化样本；W4-W6 候选规则验证 |
| **判断书"被同行选择"反馈** | 判断书生成后 → 后续是否成交 → 成交学员的真实路径 | M8 学员案例库 + M7 §九下游对账 | 验证 M7 §七 12 条规则的命中精度 / 反向修订规则集 |

**数据回流原则**：

- **不向用户索取个人信息**（除自愿留企微）——尊重 M5 §5.2 圈子重合约束
- **回流数据脱敏到子类/段（地市级不到县级、单位到子类不到具体单位名）** —— 参考 M8 学员案例的脱敏标准
- **样本流向**：小程序 session → 数据仓 → 上下文库 skill 季度跑通时归纳 → 入 M5/M7/M8 等

## 九、与上下文库的同步契约

依据：M7 §九 下游对账机制 + L3 上下文库 skill 跨 skill 调度元规则。

### 同步方向（双向）

- **上下文库 → 小程序**：M7 §七 规则集变更时，触发 06 推荐逻辑规则表 + 09 派生层更新（已在 5-18 校对通过 v0.2）→ 小程序按 06/09 自动加载，不需要前端代码改动
- **小程序 → 上下文库**：小程序 V1 上线后，每季度由 L3 跨 skill 调度报告归纳"答题分布 / Q9 命中分布 / 企微对话主题"，回写到 M5/M7/M8 候选区，由对应 L2 skill 季度跑通时拍板入库

### 不同步的情况（小程序自由度）

- 视觉、文案润色、动效、加载逻辑——由前端团队按 07-设计规范决策，不需要回上下文库
- 题目顺序微调（不改决策维度）——PRD 内部决策，不触发 M7 规则集变更

### 冲突仲裁规则

- 上下文库 vs PRD 冲突 → **上下文库优先**（M7 是判断中枢，PRD 是执行表达）
- 上下文库 vs 用户真实数据冲突 → 走 L3 季度报告 + ww 校对（不擅自修改 M7/M9 等 P0 文档）

## 十、3-6 个月演进展望

依据：经营战略总纲 §3.1.2 三层嵌套飞轮、§7.1 生存三线、§7.3 阶段共识 + 业务必读 §5.6 业务分层 ABCD。

### 3 个月内的演进重点

- **跑通 V1 主路径数据反哺**：积累 ≥1,000 答题样本 + ≥100 加企微成交转化 + ≥30 条 Q9 命中后追问的对话样本
- **R13 应答框架的有效性验证**：跑通三层追问后的成交率 vs 直接推荐的成交率对比，验证 D6 决策是否成立
- **过考率回写机制成熟**：每季度由学员案例 skill 跑通后回写 4 张过考率卡的数据，不再人工维护

### 3-6 个月的演进重点

- **判断书升级为"判断书 v2 + 陪跑地图"**：经营战略总纲 §0.4 起点承诺三层结构中"陪跑地图"是判断书的下游延伸——拿到判断书的用户应有清晰的"接下来 3-6 个月做什么"路线
- **测一测扩展到 12-15 题并引入 DP002/DP003**：M7 §八 watch 清单中 W4-W6 升 §七 后，对应应答框架扩到 dialog_protocols.json
- **小程序飞轮与上下文库飞轮的耦合度提高**：L3 季度报告产出"小程序数据反哺归纳" → 直接喂 M7 / M8 / M5 季度更新 → 反过来更新小程序，让"用户每答一题，都让研知道判断更准"成为可触达的飞轮（经营战略总纲 §3.4 内功）

### 6 个月以后的演进（不在本策划承诺范围，留作 V2 PRD 起点）

- 账号体系 + 判断书版本对照 + 同行历史选择路径相似度匹配
- 与备考服务的产品打通（小程序 → 备考方案 → 课程订单的全链路，但 ER3 不催单纪律不放松）
- 跨平台分发（公众号菜单嵌入 / 视频号小程序卡片 / 抖音组件——但不做朋友圈裂变）
