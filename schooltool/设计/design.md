## 〇 · 给同事的一页纸说明

研知道视觉系统的核心结论一句话:

> **"三层受众" × "三套语法" × "一套品牌基因",通过"母版 KV"立住,通过"派生规则"扩展到 12 个场景。**

| 层级 | 受众阶段 | 视觉策略 |
|------|----------|----------|
| **L1 冷启动** | 还不认识研知道(短视频、初次曝光) | 大色块 + 黑体 Heavy + 直白疑问句 |
| **L2 半信任** | 听过没买(详情页前 3 屏) | 米白底 + 黑体 Heavy + 强价格锚点 |
| **L3 信任** | 已决定了解(小程序 / 详情页主体 / 私域) | 衬线宋体 + 档案克制 + 编辑部口吻 |

**品牌基因(5 个,出现在所有 KV 里):**

1. 赤土橙 #CF7140 主色+ 暖灰绿 #5F8C6E 第二色(收益语义专用)
2. 8px 主色小方块签名
3. **中文刊头**:研 知 道 + 成立于 2021 年 + 川渝在职考研
4. **中文刊尾(主张)**:不骗人 · 真服务
5. 1px / 0.5px 档案线边框美学(KV/海报直角档案 · UI 场景允许圆角卡 · 见第 17 章)

**关键原则**:
- 刊头 = 身份 / 刊尾 = 主张
- **去 AI 化(详见第 13 章)** — 规范的元约束,先于一切
- **UI vs 海报分类(详见第 17 章)** — 视觉 token 统一 / 表达类型差异化(三顿半路径)
- **第二色:** 暖灰绿 #5F8C6E(详见第 14 章)

## 一 · 品牌气质

> **专业服务机构的踏实,加上服务人的温度。**

### 受众决定视觉(为什么是这个气质)

研知道的核心受众是**川渝在职考研学员**(党校在职研究生 + 199 管理类联考),以体制内、事业单位、国央企人群为主。这四个事实直接决定了视觉风格:

1. **对夸张营销话术有本能排斥** → 不用"震撼/必过/逆袭"等抓眼词,不用大色块促销视觉
2. **时间极有限** → 界面效率优先于纯美感,关键信息一屏到位
3. **决策保守** → "可信"信号必须远多于"活泼"信号,档案克制感是底色
4. **品牌承诺是"真服务"** → 视觉表达必须配得上这四个字,任何花哨都是减分

这四条也是任何视觉决策的最终裁决依据 — 当具体规则与之冲突时,回到这四条做决定。

### 三个气质坐标

| 不要 | 要 |
|------|-----|
| 度假酒店感(轻、装饰、生活方式) | 档案克制感(踏实、有据、文档级) |
| 律所网站感(冷、距离、审视) | 编辑部人格(说人话、懂你) |
| 网红考研机构(嘈杂、促销、不可信) | 老师傅说话的方式(简单但有效) |
| 政府门户(僵硬、无人格) | 像得到/丁香医生(严肃但好懂) |

### 额外反面参照(明确不像什么)

- 不像 K12 兴趣班(大字抓眼、夸张表情、彩虹色块)
- 不像 DTC 消费品(年轻化、插画化、反差萌、彩色渐变)
- 不像知识付费机构(套路化"打破认知"、紫色渐变、IP 形象)
- 不像互联网大厂(高科技感、未来感、霓虹光晕)
- 不像高奢品牌(纯黑极简、无信息量留白)

**接近什么**:得到 App 的内容卡片 + 三联中读的栏目页 + 丁香医生的功能页之间的交集。

### 自检核心问题
**"这一张图发出去,1100 个体制内学员看了,会觉得'这家像律所'还是'这家懂我'?"** —— 答案必须是后者。

### 品牌身份与主张

```
品牌身份(刊头位)   研 知 道 · 成立于 2021 年 · 川渝在职考研
品牌主张(刊尾位)   不骗人 · 真服务
业务定位            川渝党校 · MPA 双证 · 在职可读
信任锚              来自 1,100+ 川渝同行
```

**英文小标签可在内容章节里使用(如 PRODUCT NOTE / CASE),但品牌识别位必须中文**。
**注意:英文小标签使用受第 13 章"去 AI 规则"约束,一张物料最多 2 个**。

## 二 · Design Tokens

### ⚡ 2.0 复制即用 · CSS 变量块

> 给 AI 工具 / 设计师 / 开发者直接复制使用。完整 token 体系定义见下方 2.1-2.6 节。

```css
/* === 研知道视觉系统 · 完整 Token === */
:root {
  /* ---- 色彩 · 主色 + 第二色 + 底色 ---- */
  --color-accent:        #CF7140;   /* 主色 · 赤土橙(70-80% 权重) */
  --color-accent-soft:   #F1E0D3;   /* 主色浅(chip 浅底 / icon 圆背景) */
  --color-accent-dim:    rgba(207,113,64,0.08);  /* 主色透明(光晕 / hover) */
  --color-accent-deep:   #B85F38;   /* 主色深化(L1 反白 CTA 文字 / 深色态文字) */

  --color-success:       #5F8C6E;   /* 第二色 · 暖灰绿(5-10%,仅积极/收益语义) */
  --color-success-soft:  #DAE5DC;   /* 第二色浅 */

  --color-bg-base:       #F5EFE7;   /* 米白底(整页画布) */
  --color-bg-card:       #FFFFFF;   /* 卡片纯白 */
  --color-bg-tertiary:   #F0E9DD;   /* 暖米灰(分隔区域 / icon 浅背景) */

  --color-text-1:        #2A251E;   /* 主文字(偏暖黑) */
  --color-text-2:        #6B6258;   /* 副文字(暖灰) */
  --color-text-3:        #8A8175;   /* 三级文字(标签 / 元数据) */

  --color-border:        #ECE5D8;   /* 卡片边框(0.5px) */
  --color-border-strong: #2C2A26;   /* 1px 强分隔(KV/海报档案表) */
  --color-divider:       #E8E1D5;   /* 行间分割(0.5px) */
  --color-dark-bg:       #2A251E;   /* 深墨黑(CTA 反差备用) */

  /* ---- 圆角 ---- */
  --radius-thumb:     6px;     /* 缩略图 / 衍生 KV .frame 容器 */
  --radius-icon-sq:  12px;     /* 方形 icon 卡 */
  --radius-card-sm:  14px;     /* 小型卡(评价 / 推荐 / 表单内嵌) */
  --radius-card-md:  16px;     /* 标准卡(列表 / 选项 / 通知 / 海报模块) */
  --radius-card:     18px;     /* UI 默认卡(entry / dim / compare / lib) */
  --radius-card-lg:  20px;     /* 大卡(hero / 决策) */
  --radius-card-xl:  24px;     /* 反差 hero(分享预览首层 / banner) */
  --radius-circle:    50%;     /* 圆形 icon 头标 / radio */
  --radius-pill:     999px;    /* CTA 药丸 / chip(全场通用) */
  --radius-none:      0px;     /* KV/海报档案区(默认直角) */

  /* ---- 阴影 ---- */
  --shadow-card:     0 2px  8px rgba(60,50,40,0.04);   /* 标准 UI 卡片 */
  --shadow-card-lg:  0 4px 16px rgba(60,50,40,0.06);   /* 悬浮卡片 */
  --shadow-cta:      0 6px 20px rgba(207,113,64,0.22); /* 主色 CTA */

  /* ---- 字体(短名,与视觉资产 HTML 实操一致) ---- */
  --serif: "Noto Serif SC", "Source Han Serif SC", serif;
  --sans:  "Noto Sans SC",  "Source Han Sans SC",  sans-serif;
  --mono:  "JetBrains Mono", monospace;

  /* ---- 间距(8 倍数) ---- */
  --space-1:  4px;   --space-2:  8px;   --space-3: 12px;
  --space-4: 16px;   --space-5: 20px;   --space-6: 24px;
  --space-8: 32px;   --space-10: 40px;  --space-12: 48px;
}
```

**使用纪律**:
- **正文严禁写死 hex** —— 始终用 `var(--color-accent)`,改色只需改根定义
- **第二色仅用于"积极/收益"语义场景** —— 不进正文 / 标题 / CTA / 卡片底
- **CTA 永远 `--radius-pill`** —— 任何场景
- **KV/海报场景默认 `--radius-none`** + 1px 黑线 + 无阴影
- **UI/小程序场景默认 `--radius-card` (18px)**,小卡用 `--radius-card-sm/md`,hero 用 `--radius-card-lg/xl`,配 `--shadow-card` + 0.5px `--color-border`
- **衍生 KV / 缩略图容器**用 `--radius-thumb` (6px)

### ⚠️ 2.0.1 视觉资产 HTML 承载层约定(读样张代码必看)

### 2.4 圆角、阴影、渐变(分场景)

> **圆角/阴影/渐变规则按"KV/海报" vs "UI/小程序"两类分别约束**。
>
> 产品 UI 用 0 圆角硬阴影会工程师感太硬,用户不愿点;UI 圆角白卡 + 极轻阴影 + 极淡光晕,既保品味又有产品感。

#### 圆角 Radius

```
/* UI / 小程序场景(允许圆角,7 档完整)*/
Radius-thumb      6px   缩略图 / 衍生 KV .frame 容器
Radius-icon-sq   12px   方形 icon 卡(开屏页 icon 主卡可用更大,如 56px)
Radius-card-sm   14px   小型卡(评价 / 推荐 / 表单内嵌 / 模块小卡)
Radius-card-md   16px   标准卡(列表 / 选项 / 通知 / 海报模块)
Radius-card     18px    UI 默认卡(entry / dim / compare / lib 等卡内壳层)
Radius-card-lg   20px   大卡(hero / 决策)
Radius-card-xl   24px   反差 hero(分享预览首层 / banner / 大色块 hero)
Radius-circle    50%    圆形 icon 头标 / 单选 radio
Radius-pill     999px   CTA 药丸按钮 / chip 标签 / 进度条(全场通用)

/* KV / 海报场景(默认直角)*/
Radius-none     0px    KV 母版 / 详情页档案表 / 朋友圈金句卡内框 / A4 海报档案区
                       (档案克制美学)
Radius-card-sm/md  14-16px   海报内的"卡片化模块"可用(如 A4 海报的 CTA 卡 / 时间轴卡)
```

**判定原则**:
- 是给用户**点的、滑的、选的** → 用圆角(产品 UI,默认 `--radius-card` 18px,卡内小元素 14/16,hero 20/24)
- 是给用户**看的、读的** → 默认直角(KV/海报),但卡片化模块可用 14-16px

#### 阴影 Shadow

```
/* UI / 小程序场景(允许极轻阴影)*/
Shadow-card     0 2px  8px rgba(60,50,40,0.04)   标准卡片
Shadow-card-lg  0 4px 16px rgba(60,50,40,0.06)   悬浮卡片(选中态 / hero 卡)
Shadow-cta      0 6px 20px rgba(207,113,64,0.22) 主色 CTA 阴影(提示可点)

/* KV / 海报场景(默认无阴影,边框分层)*/
Shadow-none     0     KV 母版 / 详情页档案表 / 主体海报内容
                      (1px 黑线 + 0.5px 灰边框承担分层)
Shadow-card     0 2px  8px rgba(60,50,40,0.04)   海报内卡片化模块可用极轻阴影
```

**全场景禁用**:强阴影(blur > 24px 或 alpha > 0.15)、毛玻璃 backdrop-filter(L1 反白海报除外)、霓虹光晕。

#### 渐变 Gradient

```
✅ 极淡中心光晕(opacity ≤ 0.10)
   用 radial-gradient 在开屏页 / 海报背景做氛围(三顿半式柔光)
   仅主色 #CF7140 或白色透明,不引入第二色光晕

✅ L1 反白短视频封面的角落光晕
   tl rgba(255,255,255,0.10) + br rgba(0,0,0,0.10),用 radial-gradient

❌ 线性渐变(linear-gradient)做主标题 / 背景大色块 / 按钮
❌ 紫渐变、霓虹渐变、彩虹渐变(全场禁)
❌ 多色 stop 渐变 / 高饱和度渐变
```

**判定**:渐变 = 服务"氛围"(极淡光晕)合规;渐变 = 装饰主元素(标题/按钮/大色块) 禁。

## 五 · 排版黄金法则

### 5.1 三明治结构(L3 物料默认结构)

```
[刊头 - 身份位]                            
[英文小标签 或 章节小标签]                  ← 11px / 字距 3px / 主色
[中文大标题]                              ← 思源宋体 SemiBold 28-32px
[36px × 2px 主色短横线]                 
[副标题]                                  
[正文]                                    
[引句区]                                  
[P.S.]                                    
[刊尾 - 主张位]                            
```

**注意**:第 13 章去 AI 规则要求 — 不是所有 L3 物料都要严格按此结构,需要"故意打破"。

### 5.2 章节小标签法

每个内容块开头用**英文小标签 + 中文大标题反差**:

```
[REGIONAL FOCUS]    ← 11px / 字距 3px / 主色 / 英文章节小标签
精准匹配川渝院校    ← 22px / 黑体 Heavy(L1/L2)或 宋体 SemiBold(L3)
```

**重要约束(第 13 章衍生):**
- **一张物料英文小标签 ≤ 2 个**
- 仅在"内容章节分区"使用
- 长文里多个章节时,优先混合使用中文章节标识

可用英文章节标签:`PRODUCT NOTE` / `CASE` / `CONTENTS` / `P.S.` / `NOTICE` / `PROFILE`

### 5.3 引句区"主色块强突出"(L3 专属)

- 背景色 = 主色 #CF7140
- 文字 = 白色,思源宋体 Medium 18-20px
- 装饰 = 一个超大引号(48px,白色 40% 透明度)
- 署名 = "— 研知道·XX"(11px / 字距 3px / 白色 60% 透明度)

### 5.4 表格序号档案法

```
─────────────────  (1px 黑线起)
01  内容名称           备注
─ ─ ─ ─ ─ ─ ─ ─ ─  (0.5px 灰线分隔)
02  内容名称           备注
─────────────────  (1px 黑线收)
```

### 5.5 P.S. 后记法(L3 温度核心)

- 标识:`[P.S.]` 11px / 字距 3px / 主色
- 内容:思源宋体 Medium 13px,行高 1.95
- 落款:"— 研知道·编辑部" 右对齐

**重要约束(第 13 章衍生):**
- 不再使用"我们知道你 X — 这套资料的目标不是 Y,是 Z"的固定模板
- 改为"P.S. 库"(收集 30-50 句真实编辑部式短句),按场景选用
- 允许 P.S. 只有一句话(如"今晚冷,多穿件衣服。")

### 5.6 刊头/刊尾仪式法

详见第 3.3 节。注意 — 第 13 章允许"轻量物料"省略完整刊头,只保留品牌签名小方块。

## 六 · 文案语气库

### 6.1 三类文案的语气分层

| 场景 | 语气 | 示例 |
|------|------|------|
| 决策类(测评、咨询) | 沉稳、第二人称、像顾问 | "看见适合你的学历提升路径" |
| 内容类(笔记、案例) | 编辑部口吻、说实话 | "说一个反常识的话:资料不是越多越好" |
| 营销类(详情、价格) | 陪伴式、不命令 | "我准备好了,加入 2026 届" |
| 冷启动(L1) | 直白疑问、数字冲击 | "科级晋升前,要不要先把学历补上?" |

### 6.2 全局禁用词

**炒作型**:~~震撼~~ ~~爆款~~ ~~必看~~ ~~独家秘籍~~ ~~仅剩 N 名~~ ~~逆袭~~ ~~上岸秘籍~~ ~~弯道超车~~

**⚠️ 合规风险型(承诺/绝对化,广告法红线)**:~~必过~~ ~~包过~~ ~~稳过~~ ~~保过~~ ~~绝对~~ ~~100%~~ ~~零失败~~ ~~一定~~

**电商话术型**:~~亲~~ ~~宝子~~ ~~宝宝~~ ~~姐妹们~~ ~~家人们~~

**焦虑营销型**:~~再不学就晚了~~ ~~别人已经……~~ ~~内卷~~ ~~躺平~~ ~~震惊~~ ~~万万没想到~~

> ⚠️ 合规风险型尤为关键 — 教育培训行业广告法明文禁用"必过/包过/100%"等绝对化承诺,违规会面临处罚。任何文案命中此类词必须立即替换。

### 6.3 全局禁用句式
- ❌ 任何感叹号结尾的句子
- ❌ "马上抢购、立即下单、不要错过"等命令式
- ❌ "您是否还在为 X 烦恼?"等老式销售开场
- ❌ "亲""宝宝""姐妹"等电商话术

### 6.4 首选叙事结构

**问题 → 方法 → 证据 → 行动**(替代"三步逆袭 / X 个秘籍"套路):

- **问题**:明确学员当下的真实困境
- **方法**:研知道具体怎么做
- **证据**:真实案例 / 真实数据 / 1,100+ 同行经验
- **行动**:下一步可执行动作 — 不是"扫码咨询"这种空动作,而是"测一次匹配度""看课程一期目录"这种具体动作

### 6.5 时态与语态

- 优先现在时 + 肯定句
- 少用被动语态
- 少用"你会发现 / 你将收获"这种未来承诺式表达
- 第一人称称呼:**研知道 / 我们**;禁用"本公司 / 小编 / 老师我"
- 对受众称呼:**同学 / 考生 / 备考学员**;禁用"宝子 / 亲 / 家人们 / 姐妹们"

### 6.6 长度规范

| 平台 | 建议字数 |
|------|---------|
| 公众号长文 | 1,500 - 3,000 字 |
| 公众号短推送 | 500 - 800 字 |
| 抖音口播脚本 | 80 - 150 字 |
| 小红书笔记正文 | 200 - 500 字 |
| 小红书首图标题 | 10 - 16 字 |
| 短信 / 私域文案 | 50 - 100 字 |
| 朋友圈金句卡 | 单句 ≤ 28 字 |

## 十三 · 去 AI 规则(元约束)

> **这一章先于一切。** 它不是规则之一,是约束所有规则使用的"元规则"。

### 13.1 为什么需要这一章

研知道当前的视觉规范,如果**严格按规则执行**,会落入 5 个典型 AI 特征:
1. 过度对称 + 几何完美
2. "三段式"模板化结构
3. 滥用英文章节标签
4. 数字徽章成瘾
5. 范文化的 P.S. 句式

这些特征的共同根源是:**AI 完美主义 + 模板化**。

5 个特征单独看都没错,但叠加起来 = **"所有图都像同一个 AI 出的"**——这是研知道作为真实品牌的最大威胁。

**真实品牌设计师的标志,是会在合适的地方"故意打破完美"。**

### 13.2 5 条去 AI 规则

#### 规则 1 · 允许"非对称张力"

```
默认(AI 化):    所有元素居中或左右严格对齐
                 间距严格 8 倍数
                 数字徽章总是三列横排

去 AI:          ✅ 主标题左对齐 + 副标题右对齐(反差)
                 ✅ 数字徽章可"3+2"非均分
                 ✅ 关键信息可置于黄金分割点(0.618)
                 ✅ 偶尔故意"漏白"(非对称留白)
```

**触发条件**:每 5 张物料里,至少 1 张刻意打破对称。

#### 规则 2 · 允许"轻量物料"省略完整刊头

```
默认(AI 化):    每张图都"刊头 + 主体 + 刊尾"三段式
                 每段都用 1px 黑线封口

去 AI:          ✅ 主物料(详情页/小程序首页/重要海报)→ 完整结构
                 ✅ 衍生物料(社群消息/即时通知/小贴士)→ 仅签名小方块
                 ✅ 轻量物料结构示例:
                    [一句话内容]
                    
                    ■ 研知道
                 ✅ 不允许"为了规整而加刊头"
```

**判断标准**:这张图的"重要性"配得上完整刊头吗?如果不配,就只用小方块。

#### 规则 3 · 限制英文章节标签

```
默认(AI 化):    每个章节都加 PRODUCT NOTE / CONTENTS / CASE / FROM 1100+
                 字距永远 3-4px

去 AI:          ❌ 一张物料英文小标签 > 2 个 → 不允许
                 ✅ 长文里多章节时,混合使用中文章节标识:
                    "01 · 我们怎么挑选这两本"
                    "02 · 怎么用这两本"
                    "03 · 不适合的人"
                 ✅ 章节标签可以仅用数字:"01" / "02" / "03"
                 ✅ 偶尔甚至可以不用任何章节标签(章节直接用大标题)
```

**触发条件**:每 5 张物料里,至少 2 张用纯中文或纯数字章节标识。

#### 规则 4 · 限制数字徽章使用

```
默认(AI 化):    主页/详情页/小程序首页都用"1100+ / 86% / 04年 / 033法"
                 大数字徽章四列横排
                 数字下方一致用"主色 + 单位"

去 AI:          ✅ 数字徽章仅用于 L1/L2 转化场
                 ❌ L3 信任场(小程序首页/详情页主体/私域)禁用独立徽章
                 ✅ L3 改用"数据嵌入正文"叙事方式:
                    "我们带过 1,100 多位川渝在职考生……"
                    而非独立徽章 [1,100+]
                 ✅ L3 偶尔可用单一数字嵌入引句:
                    "1,100 个川渝同行,
                     做了同一个决定。"
```

**判断标准**:用户处于哪一层?L1/L2 → 徽章 / L3 → 嵌入叙事。

#### 规则 5 · P.S. 文案库

```
默认(AI 化):    所有 P.S. 都是
                "我们知道你 X —— 这套资料的目标不是 Y,是 Z"
                都用破折号
                都以"剩下的时间,留给生活"这种文艺收尾

去 AI:          ✅ 建立"P.S. 库"(目标 30-50 句真实编辑部式短句)
                 ✅ 库里允许多种风格:
                    - 短句式:"今晚冷,多穿件衣服。"
                    - 提醒式:"对了,记得周三是直播。"
                    - 反思式:"这周教材改了 3 次,抱歉。"
                    - 温度式:"我们知道你忙,所以做了这本……"(原模板,但只占库的 1/5)
                 ✅ P.S. 不再"模板化产出",而是"按场景从库里挑"
                 ❌ 禁止全文都用"我们知道你 X — Y 不是 Z"句式
```

**P.S. 库建设**:由守门人(ww)主持,每月扩充 5-10 句。

### 13.3 设计师 / AI 工具的"去 AI 自检 5 项"

每张图发出前,问自己:

```
☐ 1. 这张图有没有"刻意非对称"的元素?
     (如果完全对称完美 → 考虑加一个非对称点)

☐ 2. 这张图配得上完整刊头吗?
     (如果是轻量物料,简化为只有签名)

☐ 3. 英文小标签是否 ≤ 2 个?
     (超过就改成中文/数字)

☐ 4. L3 物料里没有独立大数字徽章?
     (如果有,改为嵌入正文)

☐ 5. P.S. 是否避免了"我们知道你 X — Y 不是 Z"句式?
     (查 P.S. 库,选不同风格)
```

5 项全过 → 通过去 AI 自检 → 这张图"看起来不像 AI 出的"。

### 13.4 这一章的检验机制

每个月做一次"AI 化体检":

1. 把当月产出的 30 张物料并排放
2. 问自己:**"这看起来像不像 30 个不同的人做的?"**
3. 如果答案是"像同一个 AI 出的 30 张" → 规范执行偏 AI 化,重新看本章

### 13.5 元原则

> **AI 不会犯错,但会让一切看起来"对得不像活的"。**
> **真实设计师的标志,是会在合适的地方"故意不对"。**
> **研知道要的不是"AI 设计师的优等生",是"懂得故意不完美的真实编辑部"。**

## 十四 · 循环色框架

> 第二色 **#5F8C6E 暖灰绿**作为"积极/收益"语义专用色已纳入品牌基因。
> 第三色(若需引入"风险态"语义)走占位流程,等品牌研究后定。

### 14.1 为什么需要循环色

研知道当前只有 1 个主色(赤土橙 #CF7140)。**长期看会出现 2 个问题**:

1. **审美疲劳**:长期物料产出后,品牌色变成"千图一色",失去新鲜感
2. **章节区分弱**:不同期数 / 不同主题 / 不同业务线在视觉上无法区分

参考 Anthropic 品牌规范的"accent 色循环"思路,研知道引入"循环色"概念:

> **品牌色锁定 1 个(赤土橙),但允许 2-3 个 accent 色作为"循环点缀",在不同场景 / 期数 / 业务线中按规则切换。**

### 14.2 循环色的使用约束(原则先确立)

无论具体哪几个色,**使用约束已经确立**:

#### 约束 1:循环色 ≠ 主色
```
赤土橙 #CF7140 永远是主色,占视觉权重 70-80%
循环色仅用于点缀(20-30%),不能喧宾夺主
```

#### 约束 2:循环色按"场景维度"使用
```
按业务线循环  党校系 vs 管综系 vs 评估系  → 各用一个 accent
按期数循环    第 1-12 期 vs 第 13-24 期    → 每年换一组
按主题循环    决策指南 vs 案例分享 vs 通知 → 各分配一色
```

**注意**:三种循环维度只能选 1 种,不能同时用 — 否则失控。

#### 约束 3:循环色只用在"特定位置"
```
✅ 引句区背景色(替代赤土橙)
✅ 章节标签颜色
✅ 头像底色 / tag 底色
❌ 不用于 CTA 按钮(CTA 永远赤土橙)
❌ 不用于主标题关键词高亮(主标永远赤土橙)
❌ 不用于刊头小方块(签名永远赤土橙)
```

#### 约束 4:循环色的"协调度"测试
```
任何候选循环色,必须满足:
- 与赤土橙 #CF7140 在同一色相家族(暖色系优先)
- 饱和度 ≤ 60(不超过赤土橙)
- 明度差 < 20(避免高对比"破画面")
- 在米白底 #F5EFE7 上有可读性
```

### 14.3 已确立的第二色

```
赤土橙 #CF7140(主色,占 70-80%)
   + 暖灰绿 #5F8C6E(第二色,占 5-10%,语义专用)
```

**第二色 #5F8C6E 暖灰绿的用法**(强制):

```
✅ 决策报告中"收益指数"进度条
✅ 决策报告中"核心收益"小卡 icon + 标题
✅ 数据可视化中"积极/上岸"语义节点(如时间轴中已完成节点的颜色变体)
✅ 案例列表中"上岸"标签
✅ 元数据栏中"LEARNERS · 1,100+"等积极数据
❌ 主标题 / 正文 / CTA / 卡片底 / 边框(任何"非积极/收益"语义场景禁用)
```

#### 14.3.1 第三色仍走原占位流程(若需要"风险态"语义)

候选方向(供未来研究参考):

```
方向 A:暖色系扩展(若需"警示态")
    赤土橙 #CF7140(主)+ 暖灰绿 #5F8C6E(收益)
    + 深酱红 #8B2E2E(用于"风险/警示"语义)

方向 B:中性扩展(若需"中立信息态")
    赤土橙 #CF7140(主)+ 暖灰绿 #5F8C6E(收益)
    + 暖灰 #8A8073(用于"中立 / 已读 / 历史"语义)
```

**判定时刻**:在出现明确"风险/警示"或"中立"场景需求时,提"提炼会"决定。

### 14.4 下一步

```
⏳ 若产品出现明确"风险态"需求(如"超时未报名提示" / "信息不全警示"),引入第三色
⏳ 8 家品牌研究若有新洞见,可考虑加"按业务线/期数循环"的扩展机制
```

## 十五 · 图像使用规范

### 15.1 人像

- **优先**:真实学员、真实老师、真实考场、真实服务场景
- **禁用**:图库网图、Avatar、AI 生成脸、夸张表情包、明星代言摆拍
- **原则**:一张真实学员照片胜过十张精修网图
- **法律前置**:任何真实学员照片必须有书面授权,授权范围必须明确(物料类型 / 投放渠道 / 时效)

### 15.2 插画

- 克制使用,默认不出现
- 若使用:简洁几何 + 单色或双色(优先赤土橙 + 米白)
- 禁用:3D 拟物插画、Q 版卡通角色、吉祥物、IP 形象、表情包

### 15.3 截图 / Social Proof

- 成绩单、证书、学员感谢截图可作为信任证据
- **必须严格匿名化**:
  - 姓名打码(可保留姓氏)
  - 学号 / 准考证号全部打码
  - 联系方式 / 微信号 / 手机号全部打码
  - 工作单位若涉及部委 / 厅局 / 国企必须打码
- **禁止**:
  - 假截图 / Photoshop 合成
  - 截图本身做成"营销海报"(违和感强)

### 15.4 禁用图像类型(高风险)

- 夸张表情包("你还在混日子吗"类)
- 焦虑营销图(倒计时 + 红字 + 惊叹号组合)
- 大面积滤镜美颜照
- 拼贴式电商详情页(多元素堆叠 / 重叠彩字)
- AI 生图特征明显的合成图(六指 / 不自然光影 / 透视错乱)

## 十六 · 动效原则

主要适用于小程序 UI、H5 落地页、视频转场。**静态物料(详情页 / 海报 / 图文)默认无动效**。

### 16.1 基调

温和、快速、不抢视线。**动效服务于任务指引(引导注意力到下一步),不是装饰**。

### 16.2 时长

- 默认过渡:200 - 300ms
- 上限:不超过 500ms
- 入场:200ms / 离场:150ms

### 16.3 类型

- **优先**:`fade`(透明度) / `slide`(位移)
- **避免**:`bounce`(弹跳) / `spin`(旋转) / `zoom`(缩放震动)
- **禁用**:粒子特效、闪烁、霓虹光晕动画、3D 翻转

### 16.4 缓动

- 默认 `ease-out`
- CTA 按钮按下:`cubic-bezier(.2,.8,.2,1)` 自然弹性,但幅度 ≤ 4%
- 禁用 `bounce` / 强弹性曲线

### 16.5 自检 2 问

- 这个动效如果**取消**,任务还能完成吗? → 能 → 这个动效是装饰,删掉
- 这个动效**第二次看**还想看吗? → 不想 → 时长太长 / 类型太重

## 十七 · UI vs 海报统一视觉系统

> 三顿半路径:**视觉 token 完全统一,表达类型分两类差异化处理**。
>
> 17.1 节定义"UI 类"和"海报类"的边界;17.2-17.4 定义两类各自的差异化规则;17.5 验证清单。

### 17.1 两类物料的定义

```
UI 类(用户在手机里完成任务):
   小程序页面 / H5 测评工具 / 淘宝详情页 / 私域小工具
   特征:用户要点击、选择、滑动、完成一个流程
   设计目标:规则感 + 清晰感 + 互动感

海报类(用户瞄一眼是否停留):
   朋友圈金句 / 小红书图文 / 公众号封面 / 抖音/视频号封面 / A4 线下传单 / 微信群通知
   特征:用户在 feed 流中刷过,要在 0.5 秒内决定是否停留
   设计目标:创意感 + 新鲜感 + 视觉冲击力
```

**判定**:产物**要不要点**? → 要 = UI;不要 = 海报。

### 17.2 共用 Token(全场强制一致)

所有 UI 和海报必须共用同一套 token——这是"系统统一性"的根基。任何 token 偏离都会让品牌识别度下降。

| Token 类型 | 共用值 |
|---|---|
| 主色 | #CF7140 赤土橙(70-80%) |
| 第二色 | #5F8C6E 暖灰绿(5-10%,仅积极/收益) |
| 底色 | #F5EFE7 米白 |
| 卡片底 | #FFFFFF 白 |
| 字体策略 | 衬线 hero / sans UI(同 §2.2) |
| 中英文半角空格 | 全场强制 |
| 章节标识 | 优先纯中文 / 数字,英文小标签 ≤ 2 个/张(同 §13 规则 3) |
| CTA 形式 | pill 药丸按钮 + 主色填充 + 白字 + 右箭头 |
| 主张刊尾 | "不骗人 · 真服务"(海报必带,UI 可省) |

### 17.3 UI 类差异化规则

```
✅ 圆角 14-24px 白卡(小卡 14/16px / UI 默认 18px / hero 20-24px)
✅ 极轻阴影 Shadow-card 0 2px 8px(配合 0.5px 边框分层)
✅ 圆形 icon 头标(承担"这张卡是什么"的提示)
✅ 进度条(流程指示 / 数据可视化)是核心组件
✅ Tab Bar(底部导航)
✅ chip 标签(横向筛选栏 / 卡内元数据)
✅ 表格式细节(左标签灰 / 右数据黑,灰线分隔)
✅ 顶部进度数字"X / N" + 顶部进度条

❌ 大字 hero(超过 48px 的标题不在 UI 出现,UI 主标题 24-32px 衬线)
❌ 满底反白(UI 默认米白底 + 白卡,不做主色满底)
❌ 极淡光晕(UI 默认无光晕,保持任务焦点清晰)
❌ 装饰性几何元素(无背景几何图、无装饰线)
```

### 17.4 海报类差异化规则

```
✅ 大字 hero(主标题可 60-116px 衬线宋体)
✅ 极淡中心光晕 opacity ≤ 0.10(三顿半式柔光,氛围用)
✅ 关键数据可视化作"视觉锚"(如 A4 海报的 4 节点时间轴 HUD)
✅ L1 反白海报允许主色满底 + 米白字反白(短视频封面 9:16)
✅ 衬线大字配赤土橙关键词高亮("最后八周,把考点织成一张网")
✅ 期号 / 章节标 / 刊尾 等"编辑部出品"信号(轻量海报可省刊头)
✅ 几何留白:海报允许大面积留白做呼吸,UI 不允许(UI 要信息密度)

❌ Tab Bar / 顶部进度条 等 UI 控件(海报不是工具)
❌ 流程类组件(单选卡 / radio / 表单输入)不进海报
❌ 横向筛选 chip(海报没有"切换"动作)
```

### 17.5 验证清单(每张物料发出前自检)

```
🔍 这是 UI 还是海报?(先判断类型)
   要点的 → UI
   要看的 → 海报

🔍 共用 token 是否一致?(强制)
   □ 主色用了 #CF7140 还是别的?
   □ 第二色 #5F8C6E 是否只出现在"积极/收益"语义场景?
   □ 字体策略对吗?(衬线 hero / sans UI)
   □ 中英文之间半角空格了吗?

🔍 类型规则是否对位?
   UI:
   □ 圆角白卡 14-24px(默认 18px,卡内小元素 14/16,hero 20/24)?
   □ 极轻阴影 + 边框?
   □ 没有大字 hero / 满底反白 / 装饰光晕?

   海报:
   □ 衬线大字 hero?
   □ 极淡光晕仅作氛围?
   □ 没有 Tab Bar / 单选卡等 UI 控件?

🔍 通过第 13 章去 AI 检查?(元约束)
🔍 通过第 9 章 9 步检查清单?
```

### 17.6 速查矩阵

| 元素 | UI | 海报 |
|---|---|---|
| 圆角 | 14-24px 卡(默认 18px)+ pill CTA | 直角档案区 + 14-16px 模块卡 |
| 阴影 | 0 2px 8px rgba(60,50,40,0.04) | 默认无(模块卡可有极轻阴影) |
| 渐变 | 默认无 | 极淡中心光晕 opacity ≤ 0.10 |
| 主标字号 | 24-32px 衬线 | 60-116px 衬线 |
| 数据可视化 | 进度条 + chip + 表格 | 时间轴 HUD + 数据节点 |
| 章节标识 | 顶部"X / N"+ 进度条 | 期号 + 章节小标 |
| 主张刊尾 | 可省 | 必带 |
| 反白态 | 不允许 | L1 强冲击场允许(短视频封面) |
