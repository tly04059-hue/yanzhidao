## 3. L1 标准化层关键字段

L1 目标是“完整、稳定、可追溯”，不直接面向前台。

核心字段包括：

- 稳定 ID：`record_id`、`school_id`、`program_id`、`track_id`
- 学校信息：`school_name`、`school_code`、`province`、`city`、`school_level`
- 项目信息：`program_type`、`degree_type`、`exam_type`、`department`、`major_name`、`direction`
- 学习信息：`study_mode`、`duration`、`tuition_min`、`tuition_max`、`class_time`、`class_location`
- 招录信息：`enrollment`、`exam_subjects`、`latest_score`、`admission`、`admission_rate`、`admission_analysis`
- 展示信息：`description`、`tags`、`website`、`logo_url`
- 来源与质量：`source`、`quality`

### 3.1 编码、名称、展示文本必须同时保留

前台可以展示合并文本，但 L1 标准化 JSON 不能只保留合并文本，必须同时保留 `code`、`name`、`label`。

| 对象 | L1 字段 |
| --- | --- |
| 学校 | `school_code`、`school_name`、`school_label` |
| 院系 | `department_code`、`department_name`、`department_label` |
| 专业类别 | `major_category_code`、`major_category_name`、`major_category_label` |
| 专业 | `major_code`、`major_name`、`major_label` |

例如前台可展示 `(10610)四川大学`，但标准化层必须保留 `school_code = 10610`、`school_name = 四川大学`、`school_label = (10610)四川大学`。

### 3.2 管综字段表补充规则

`schoolData/sourceData/199exam/管综院校字段表.xlsx` 已确认为管综 MPA/MEM/MBA 字段标准之一。

- A-Z 列是一阶段字段，也是小程序全年展示字段。
- 管综数据最小颗粒度为“招生项目/专业方向/学习方式”，不是学校。
- 展示顺序为 `MPA -> MEM -> MBA`。
- 筛选优先级为 `项目 -> 所在省 -> 所在市`。
- `研究方向` 不包含“全日制”或“非全日制”，学习方式单独写入 `study_mode`。
- 如果调剂录取分析中的 `名额` 为正数，`adjustment` 应为 `是`。

## 4. L2 发布层关键字段

L2 目标是“小程序可直接读取”，只保留已审核、完整度达标、适合前台展示的数据。

一期必需字段：

- `school_name`
- `program_type`
- `province`
- `city`
- `tuition_min` / `tuition_max`
- `duration`
- `study_mode`
- `class_time`
- `latest_score`
- `description`
- `tags`
- `filter_tags`
- `sort_signals`
- `publish`

## 5. 完整度判断建议

一期 `publish_ready = true` 建议满足：

1. 必需字段完整度 >= 80%。
2. 学费、学制、上课方式、最新分数线至少有 3 项完整。
3. 有可展示的 `description` 或 `admission_analysis`。
4. 来源文件和批次可追溯。
5. 审核状态为 `reviewed` 或 `published`。
