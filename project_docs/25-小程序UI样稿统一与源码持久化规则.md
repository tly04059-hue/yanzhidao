## 2. 样稿视觉口径

小程序一期应统一为“温暖、克制、咨询工具型”的风格，不做营销页化。

核心视觉规则：

- 页面底色：`#F5EFE7`
- 卡片底色：`#FFFFFF`
- 柔和底色：`#F0E9DD`
- 正文主色：`#2A251E`
- 次级文字：`#6B6258`
- 提示文字：`#8A8175`
- 主操作色：`#CF7140`
- 主操作浅底：`#F1E0D3`
- 辅助绿色：`#5F8C6E`
- 辅助绿色浅底：`#DAE5DC`
- 分割线：`#E8E1D5`
- 边框：`#ECE5D8`
- 卡片阴影：`0 2px 8px rgba(60, 50, 40, 0.04)`
- CTA 阴影：`0 6px 20px rgba(207, 113, 64, 0.22)`

字体规则：

- 正文：`14px`
- 提示性小字：`12px`
- 标题：`16px`，`font-weight: 700`
- 底部导航文字：`12px`
- 不使用负字距；当前小程序统一 `letter-spacing: 0`

## 4. 后续修改规则

1. 视觉样式优先改 `miniapp/src/styles/v3.scss` 或对应 `miniapp/src/pages/**/index.vue`，不要直接改 `miniapp/dist/` 作为长期方案。
2. `miniapp/dist/` 是构建产物，可以用于微信开发者工具预览，但会被下一次构建覆盖。
3. 页面级样式如果和全局规则冲突，应先判断是否属于样稿特殊页面；否则尽量向全局规则收口。
4. 每次改完小程序样式后，至少执行：

```bash
npm run type-check
python3 projectOps/scripts/build-miniapp.py --target h5
python3 projectOps/scripts/build-miniapp.py --target mp-weixin
npm run format:mp-pages
python3 projectOps/scripts/check-all.py
```

5. H5 浏览器预览地址通常是：

```text
http://localhost:5173/#/
```

6. 微信开发者工具应打开：

```text
miniapp/dist/dev/mp-weixin
```

## 5. 预览说明

如果本地 H5 开发服务正在运行，修改 `miniapp/src` 后，浏览器刷新或热更新即可看到效果。

微信开发者工具看到的是 `dist/dev/mp-weixin` 构建结果；因此源码改完后，需要运行 `mp-weixin` 构建，让 `dist` 重新生成，开发者工具才能看到最新效果。
