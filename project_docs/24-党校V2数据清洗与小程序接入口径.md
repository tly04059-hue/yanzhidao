# 党校 V2 数据清洗与小程序接入口径

> 更新时间：2026-05-18  
> 来源：`schoolData/sourceData/party/四川党校在职研究生信息库V2.xlsx`、`schoolData/sourceData/party/重庆党校在职研究生信息库V2.xlsx`  
> 输出：`schoolData/standardized/party/party_school_l1_v2.json`、`schoolData/publish/party_school_miniapp_publish.json`、`schoolData/dx_data.json`、`miniapp/src/data/party-school-miniapp-publish.json`

## 1. 数据层级

党校 V2 数据按 L1/L2 分层处理：

- L1：完整信息库，保留院校核心、专业详情、历年报录、分数线、FAQ、特殊政策、川渝党校对比。
- L2：小程序发布库，只放前台展示需要的核心字段。
- `dx_data.json`：内部 HTML 院校库页面使用，保留完整分板块内容，同时提供 `frontend` 核心展示字段。

本次清洗生成 8 条 L1/L2 专业记录：

- 四川党校：经济学、政治学、法学。
- 重庆党校：经济管理、法律、公共管理、党政管理、战略管理。

## 2. 前台展示口径

小程序和内部院校库优先展示：

- 招生专业、专业培养方向。
- 招生总人数、各专业人数。
- 最新录取分数线。
- 报考录取率、剔除弃考实考录取率。
- 学费、学制、上课地点、上课方式。
- 报考条件摘要：党员要求、学历、工作年限。

FAQ、特殊政策、历年深度数据、参考书目、岗位适配等不全部放到前台卡片，但必须保留在信息库里，后续给 AI 问答、推荐策略和顾问使用。

## 3. 党校与管综对比口径

党校与 MPA/MBA/MEM 对比时，共用这些一级字段：

- `system`
- `program_type`
- `school_code`
- `school_name`
- `school_label`
- `province`
- `city`
- `degree_type`
- `exam_type`
- `tuition_total`
- `duration_years`
- `study_mode`
- `class_time`
- `class_location`
- `latest_score`

党校特有但很关键的字段：

- `party_member_required`
- `work_years_required`
- `eligible_regions`
- `national_exam_required`
- `admission_rate_latest`
- `actual_exam_rate_latest`

不要为了前台方便删除这些字段，否则后续无法做党校 vs MPA、四川党校 vs 重庆党校、推荐策略方案匹配。

## 4. 推荐策略更新

推荐方案库已新增/细化党校策略：

- `strategy_dx_in_service`：体制内硬条件补齐、预算谨慎、先确认单位认可。
- `strategy_dx_chongqing_open_entry`：重庆党校低门槛路径，不限党员，2 年工作经验。
- `strategy_dx_sichuan_party_member_ethnic`：四川党校党员干部路径，重点关注党员身份、3 年工作经验、民族专项和加分政策。

测一测仍然是匹配预生成策略，不实时生成全新方案。

## 5. 时间节点更新

小程序备考方案页已支持两套节奏：

- 管综：6-12 月，基础、强化、冲刺、联考。
- 党校：3-6 月，资格确认、报名材料、专业课强化、考试录取跟进。

党校路径的关键风险不只是考试难度，而是资格、单位认可、专业选择和资格审查材料。

## 6. 学生案例口径

当前学生案例库已有党校系案例。现阶段不新增案例筛选能力，但推荐结果和时间线页面可以引用“党校系同行案例”口径，帮助用户理解：

- 为什么选择党校。
- 选择四川还是重庆。
- 是否适合继续比较 MPA 双证。

后续案例库更新时，应按系统、目标、学历起点、年龄段、路径类型继续补标签。

## 7. 更新脚本

党校 V2 数据由以下脚本生成：

```bash
python3 AIAgent/scripts/build_party_school_v2.py
```

中控流水线已接入：

```bash
python3 projectOps/scripts/update-all.py
python3 projectOps/scripts/sync-miniapp-data.py
python3 projectOps/scripts/check-all.py
```

后续更新两个 V2 Excel 后，优先跑中控流水线，不要手工改小程序发布 JSON。
