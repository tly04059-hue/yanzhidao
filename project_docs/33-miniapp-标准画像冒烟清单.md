# miniapp 标准画像冒烟清单

更新时间：2026-06-01

## 目的

这份清单用于回归检查 `测一测 -> loading -> 结果页 -> 保存长图页` 是否同口径、同推荐、同案例、同解释，避免出现“首推是 A、故事却像 B、长图又在说 C”的牛头不对马嘴问题。

本轮清单基于以下真实运行链路生成：

- 统一题库：`miniapp/src/data/v5/quiz.ts`
- 统一归一化：`miniapp/src/data/quiz-normalization.ts`
- 统一总装配：`miniapp/src/data/quiz-runtime.ts`
- 统一推荐规则：`miniapp/src/data/recommendation-strategy.ts`
- 正式案例与同行数据：
  - `schooltool/data/cases.json`
  - `schooltool/data/peer_profile_cache.json`
  - `schooltool/data/students_map.json`
- 统一结果渲染：
  - `miniapp/src/pages/result/index.vue`
  - `miniapp/src/pages/zexiao/index.vue`

本轮真实数据由脚本 `miniapp/scripts/smoke-scenarios.ts` 跑出。

## 使用方式

每次调整题目口径、推荐规则、案例匹配、同行对比、长图文案后，都至少跑这 5 组画像。

每组画像按下面 5 项核对：

1. 测一测答案是否落到了预期画像。
2. 结果页首推路径、备选路径、院校名称是否合理。
3. 结果页相似案例、同行对比、真实作用是否与首推路径同口径。
4. 长图页首推、备选、相似故事、解释文案是否与结果页一致。
5. 是否存在“推荐党校，但案例在讲 MBA / MPA”或“推荐双证，但长图在讲党校内部使用”的错位。

## 通过标准

- 通过：测一测画像、结果页、长图页三者口径一致。
- 基本通过：主推荐正确，但案例或同行文案还有轻微泛化。
- 不通过：主推荐、案例、长图解释明显不一致。

- 地区：四川
- 学历：全日制本科
- 年龄：31-35
- 目标：防御/提升下限
- 岗位：财政发改
- 预算：5-8w
- 顾虑：英语不好

### 预期

- 主线应偏党校在职研究生路径。
- 备选可给 MPA 双证。
- 相似案例应优先展示四川党校/党政机关/体制内内部使用场景。

### 当前真实输出

- 规则：`P001`
- 首推：`四川省委党校 · 经济学`
- 备选：`西南财经大学 · MPA`
- 相似案例：`吴同学 · 四川党校 · 政治学`

### 结果页核对点

- 首推、备选、风险提示、每周动作一致。
- 同行对比主柱为 `四川党校`，非全为次柱。
- 相似故事前 3 条已回到党校方向，不再先冒出 MPA。

### 长图页核对点

- 标题、备选、推荐理由与结果页一致。
- `pathMeaning` 已明确解释为“体制内内部使用 + 成本敏感 + 不想押英数”。
- 长图故事顺序与结果页一致。

### 结论

- 状态：`通过`
- 备注：这是当前最稳定的一组基准画像。

- 备选：`重庆大学 · MPA`
- 相似案例：`王同学 · 西南医科大学 · MPA`

### 结果页核对点

- 主推荐方向正确，风险提示也已经回到“职称更看双证通用性”。
- 同行对比人数和路径已经能跑通。
- 相似案例已进一步从泛化 `医科类院校 · MPA` 收紧到 `西南医科大学 · MPA`。
- 故事池前两条都已回到 `医科类院校 / 西南医科大学` 这类更贴医疗语义的双证样本。
- 当前虽然还没有真正的 `MEM` 上岸案例，但至少不会再出现“主推 MEM，第一条故事却先讲国企 MBA”的错位。

### 长图页核对点

- 长图首推/备选与结果页一致。
- `pathMeaning` 逻辑方向正确，已说明“双证可查 + 职称/跨系统”。
- 长图引用案例已回到 `西南医科大学 · MPA` 语义，和结果页一致。
- 若无真实 MEM 样本，会明确提示“当前使用医科类 MPA / 医护双证案例作为最近似参考”。

### 结论

- 状态：`通过`
- 备注：当前已做到“主推 MEM，案例至少先讲医科/MPA 双证”；后续若补到真实 MEM 案例，可继续增强语义贴脸度。
