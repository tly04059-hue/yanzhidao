# 管综 L1 标准化数据结构与清洗映射表

> 更新时间：2026-05-14  
> 来源文件：`schoolData/sourceData/199exam/管综院校字段表.xlsx`  
> 本阶段目标：设计正式的管综院校库 L1 标准化数据文件结构和清洗映射表，不迁移真实全量数据，不影响小程序现有展示。

| 路径 | 用途 |

| `schoolData/standardized/199exam/README.md` | 管综 L1 标准化目录说明和文件约定。 |

## 2. L1 数据文件建议

后续正式清洗后的主文件建议放在：

```text
schoolData/standardized/199exam/199exam_school_program_master.json
```

建议文件结构：

```json
{
  "metadata": {
    "dataset": "199exam_school_program_master",
    "version": "2026-05-initial",
    "source_files": [
      "schoolData/sourceData/199exam/管综院校字段表.xlsx"
    ],
    "record_grain": "program_school_department_major_direction_study_mode",
    "updated_at": "2026-05-14"
  },
  "records": []
}
```

正式数据不要以学校为唯一粒度，而是按以下粒度拆记录：

```text
项目 -> 学校 -> 院系 -> 专业类别 -> 专业 -> 研究方向 -> 学习方式
```

## 3. 单条记录结构

单条记录分为 8 个部分：

| 模块 | 内容 |
| --- | --- |
| 稳定 ID | `record_id`、`school_id`、`program_id`、`track_id` |
| 基础定位 | `program_type`、`province`、`city`、`study_mode` |
| code/name/label | 学校、院系、专业类别、专业都必须同时保留拆分字段和展示字段；研究方向如果带编号，也保留 `direction_code` 和 `direction_label` |
| 项目核心信息 | 学制、学费、招生人数、上课地点、上课方式、大专报考、专项计划 |
| 最新分数与录取 | 最新分数线、复试人数、录取人数、录取率、是否调剂 |
| 年度复试考情 | `retest_info_by_year` |
| 年度一志愿录取分析 | `first_choice_admission_by_year` |
| 年度调剂录取分析 | `adjustment_admission_by_year` |

## 4. 一阶段字段完整度

一阶段完整度建议围绕 A-Z/AA 计算。

建议必填字段：

- `program_type`
- `province`
- `city`
- `study_mode`
- `duration`
- `tuition_min`
- `school_code`
- `school_name`
- `school_label`
- `school_level_display`
- `department_code`
- `department_name`
- `department_label`
- `major_category_code`
- `major_category_name`
- `major_category_label`
- `major_code`
- `major_name`
- `major_label`
- `direction_code`
- `direction`
- `direction_label`
- `enrollment`
- `latest_score`
- `class_location`
- `class_time`
- `adjustment`

一期进入小程序发布层建议：

1. 一阶段字段完整度 >= 80%。
2. 学费、学制、招生人数、分数线、上课方式至少 4 项完整。
3. 来源文件、sheet、行号、批次完整。
4. `review_status` 为 `reviewed` 或 `published`。

## 5. 清洗转换规则

| 规则 | 说明 |
| --- | --- |
| 合并文本拆分 | `(10610)四川大学` 拆为 `code=10610`、`name=四川大学`、`label=(10610)四川大学`。 |
| label 保留 | 前台可展示 `label`，但 L1 不能只保留 `label`。 |
| 学费换算 | Excel 中 `总学费（万）` 统一乘以 10000，落为元。 |
| 院校性质 | L1 保留原值；发布层可映射为 `985`、`211`、`双非`。 |
| 研究方向 | 去除“全日制”“非全日制”等学习方式字样，学习方式放入 `study_mode`。 |
| 最新分数线 | `P/Q/R` 拼为 `latest_score`，同时保留 `total_score`、`english_score`、`comprehensive_score`。 |
| 是否调剂 | 优先使用 X 列；若调剂年度分析任一年度 `quota > 0`，应推导为 `是`。 |
| 年度字段 | 复试考情、一志愿录取、调剂录取均按年份嵌套保存。 |

## 6. 映射表使用方式

`schoolData/mappings/199exam_field_mapping.json` 是后续清洗脚本的依据。

建议清洗脚本流程：

1. 读取 Excel 第 6 行开始的数据行。
2. 按映射表把列值写入 L1 字段。
3. 对 code/name/label 字段执行拆分。
4. 对学费、分数、录取率、调剂名额执行类型转换。
5. 生成 `record_id`、`program_id`、`track_id`。
6. 计算一阶段完整度和缺失字段。
7. 输出到 `schoolData/standardized/199exam/199exam_school_program_master.json`。

## 7. 下一步建议

确认本结构后，下一步可以做一个小样本清洗脚本：

1. 从 `管综院校字段表.xlsx` 的示例行提取 5 条记录。
2. 输出到临时样例文件。
3. 校验 code/name/label、学费换算、年度字段嵌套是否正确。
4. 再决定是否接入真实全量院校表。
