# 管综样本清洗脚本与验证结果

> 更新时间：2026-05-14  
> 目标：从 `管综院校字段表.xlsx` 示例行提取 5 条 L1 标准化记录，验证字段拆分、学费换算、年度嵌套和完整度计算。

## 1. 新增脚本

| 路径 | 用途 |
| --- | --- |
| `AIAgent/scripts/extract_199exam_sample.py` | 从管综字段表提取 5 条样例记录。 |
| `AIAgent/scripts/README.md` | 记录 AIAgent 脚本用途。 |

脚本只使用 Python 标准库解析 xlsx，不依赖 `openpyxl`。

默认命令：

```bash
python3 AIAgent/scripts/extract_199exam_sample.py
```

## 2. 输出文件

| 路径 | 说明 |
| --- | --- |
| `schoolData/standardized/199exam/199exam_school_program_master.sample.json` | 5 条样例 L1 标准化记录，仅用于验证，不是前台发布数据。 |

输出结构：

```json
{
  "metadata": {},
  "records": [],
  "validation": {}
}
```

## 3. 已验证内容

脚本已验证：

- 学校、院系、专业类别、专业均保留 `code/name/label`。
- 研究方向如果带编号，保留 `direction_code`、`direction`、`direction_label`。
- 学费从“万”换算为“元”，例如 `8.4` 转为 `84000`。
- 复试考情、一志愿录取分析、调剂录取分析按年度嵌套。
- 一阶段完整度按核心字段计算。
- 样例记录保留来源文件、sheet、行号和批次。

## 5. 注意事项

1. 样例文件来自字段表中的示例行，不代表正式全量院校库。
2. 当前 `publish_ready` 仍为 `false`，因为样例记录未经人工审核。
3. 第 7-10 行虽然完整度超过 80%，但缺少分数线、上课地点、上课方式、是否调剂，不能直接视为前台最终可发布数据。
4. 后续接入真实全量表前，应先确认真实表字段能否和本映射表稳定对齐。

## 6. 下一步建议

下一步可以基于这个脚本做正式清洗脚本：

1. 读取真实全量管综院校数据源。
2. 复用 `199exam_field_mapping.json`。
3. 输出正式 L1 主文件 `199exam_school_program_master.json`。
4. 生成完整度报告，标出低于 80% 的学校/专业方向。
5. 再从 L1 筛选生成小程序 L2 发布数据。
