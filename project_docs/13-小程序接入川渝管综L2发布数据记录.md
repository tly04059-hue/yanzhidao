## 1. 本次接入内容

- `miniapp/src/pages/school-detail/index.vue`

## 2. 前台展示颗粒度

L2 原始发布记录为 103 条，颗粒度来自 L1：

```text
学校 -> 项目类型 -> 院系 -> 专业 -> 方向 -> 学习方式
```

小程序列表页不直接铺开 103 条，而是按：

```text
学校 + 项目类型
```

聚合展示，共生成 35 个院校项目卡片。

详情页展示该学校项目下的多条项目记录，保留院系、专业、方向、学习方式、学费、学制、复试线、招生人数等核心字段。

## 4. 校验结果

已完成：

1. `npm run type-check` 通过。
2. `npm run build:h5` 通过。
3. H5 本地访问 `http://localhost:5173/#/pages/schools/index` 可打开院校列表。
4. 列表页渲染 35 个院校项目卡片。
5. 院校详情页可打开，并展示 L2 中的招生人数、复试线、是否调剂、上课地点、项目记录。
6. MBA 筛选可用，筛选后展示 11 个院校项目卡片。

构建中出现 Sass 旧 API 和 `@import` 的 deprecation warning，属于现有依赖/样式写法警告，不阻塞本次接入。

## 5. 后续注意事项

1. 后续 MEM 到位后，应先进入 L1，再生成同结构 L2，最后同步到小程序数据文件。
2. 后续党校数据建议单独进入党校 L1/L2，再在小程序适配层合并展示。
3. 小程序前台仍只展示核心字段，完整复试和录取分析继续留在 L1。
4. 如果未来要把 35 个卡片改成“26 所学校”维度，需要重做列表聚合口径，但详情页仍应保留项目记录。
