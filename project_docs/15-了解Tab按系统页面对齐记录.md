# 了解 Tab 按系统页面对齐记录

> 更新时间：2026-05-14  
> 参考样稿：`schooltool/docs/ui_samples/06_浏览详情_按系统.html`、`02_了解Tab.html`  
> 修改范围：`miniapp/src/pages/learn/index.vue`、`miniapp/src/pages/schools/index.vue`

## 1. 本次目标

把“了解 Tab”补成用户犹豫期可以浏览的路径入口，让用户能从三条路径进入：

1. 党校系
2. 管综系（MPA / MBA / MEM）
3. 同等学力申硕

每条路径都提供两个动作：

1. 看院校
2. 测一测

## 2. 页面调整

### 了解 Tab

文件：`miniapp/src/pages/learn/index.vue`

已加入：

1. 三系对比模块。
2. 每个系统的标签、学费、学制、学习模式和解释说明。
3. “看院校”和“测一测”按钮。
4. “怎么选？”建议模块。
5. 院校库描述改为读取当前 L2 数据聚合结果，显示当前已接入川渝 MPA/MBA 数据。

当前默认展开“按系统”，更符合用户进入了解页后的第一层认知顺序。

### 院校列表页

文件：`miniapp/src/pages/schools/index.vue`

已加入：

1. 支持 URL 参数 `filter`，例如从了解页进入：`/pages/schools/index?filter=管综系`。
2. 筛选参数兼容 H5 二次编码。
3. 新增 `同等学力` chip。
4. 新增空状态：当某条路径数据尚未接入时，显示“数据整理中”，并提供“先测一测”按钮。

重要口径：

- `党校系` 不再因为学校名称包含“党校”就展示 MPA 数据。
- 当前党校 L2 尚未并入，因此党校系院校列表显示“数据整理中”。
- 中共四川省委党校、中共重庆市委党校的 MPA 仍属于管综 MPA 数据，不混入党校系路径。

## 3. 验证结果

已完成：

1. `npm run type-check` 通过。
2. `npm run build:h5` 通过。
3. 浏览器访问 `http://localhost:5173/#/pages/learn/index` 正常。
4. 页面可见“三系对比”“党校系”“管综系（MPA / MBA / MEM）”“同等学力申硕”“怎么选？”。
5. 管综系进入院校库后，展示 `26 所院校 · 35 个项目`。
6. 党校系进入院校库后，显示 `党校系数据整理中`，不混展示 MPA 数据。

构建 warning：

- Sass legacy JS API warning。
- Sass `@import` deprecation warning。
- uni-app 新版本提示。

以上不阻塞本次页面对齐。

## 4. 后续建议

1. 党校 L2 发布数据生成后，将 `党校系` chip 接入正式党校院校库。
2. MEM 数据补齐后，继续并入管综系。
3. 同等学力数据如果后续要展示，需要建立单独 L1/L2 结构，避免和管综字段混用。
4. 下一步可继续对齐 `07_路径对比页.html`，让“路径对比”页真正支持党校系、MPA、同等学力的多维比较。
