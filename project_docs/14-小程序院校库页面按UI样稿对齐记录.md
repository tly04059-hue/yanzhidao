# 小程序院校库页面按 UI 样稿对齐记录

> 更新时间：2026-05-14  
> 参考样稿：`schooltool/docs/ui_samples/05_院校列表页.html`、`08_院校详情页.html`、`06_浏览详情_按系统.html`  
> 修改范围：`miniapp/`，`schooltool/` 仅读取未修改。

## 1. 本次对齐目标

老板样稿中的院校库页面核心结构为：

1. 顶部 sticky 导航。
2. 搜索栏。
3. 横向 chip 筛选。
4. 排序条。
5. 院校卡片：匹配度、标签、学费、学制/上课等核心指标。
6. 院校详情：Hero、五维评估、招生信息、同行案例、编辑部观察、底部 CTA。

本次按样稿结构保留页面视觉和交互骨架，同时改为使用川渝 MPA/MBA L2 真实数据渲染。

## 2. 页面调整

### 院校列表页

文件：`miniapp/src/pages/schools/index.vue`

已调整：

1. 筛选 chip 改为更接近样稿和业务路径的口径：`全部`、`党校系`、`管综系`、`MPA`、`MBA`、`川渝本地`。
2. 数量展示改为：`26 所院校 · 35 个项目`，避免只说“35 个院校项目”造成误解。
3. 院校卡片继续沿用样稿结构，使用真实 L2 数据展示：
   - 学费
   - 最新复试线
   - 招生人数
   - 项目类型
   - 学习方式摘要
   - 匹配度

### 院校详情页

文件：`miniapp/src/pages/school-detail/index.vue`

已调整：

1. 招生信息增加老板要求的核心字段：
   - 招生人数
   - 最新复试线
   - 学费 / 学制
   - 上课方式
   - 是否调剂
   - 退役 / 少骨计划
   - 上课地点
   - 项目记录数
2. 项目记录从最多展示 4 条改为展示全部 L2 项目记录。
3. 每条项目记录展示：
   - 招生专业
   - 院系
   - 专业培养方向
   - 学习方式
   - 学费
   - 学制
   - 复试线
   - 招生人数
   - 上课方式
   - 录取摘要
4. 同行案例板块保留样稿位置，但因真实学生案例尚未接入，改为“案例库整理中”，不再展示占位假案例。
5. 匹配度改为读取 L2 适配层计算结果，不再固定 92%。

## 5. 后续建议

1. 学生案例数据准备好后，优先接入详情页“这所院校的同行”板块。
2. MEM 数据到位后，沿用当前 L2 适配结构并入列表和详情。

4. 如果老板要求完全按 `06_浏览详情_按系统.html` 做“按系统了解”页面，下一步应改造 `miniapp/src/pages/learn/index.vue` 或新增系统详情页。
